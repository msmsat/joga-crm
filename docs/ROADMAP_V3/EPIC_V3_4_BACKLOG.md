## Фаза 1 — График работы филиала

### 1. Бэк: дефолтный график при создании + upsert при правке + бэкфилл · 🟡 · 1:30
**Простыми словами:** у каждого филиала всегда есть график (владелец сказал: «не обязательный, но по умолчанию — есть»). Создание филиала автоматически заводит 7 строк; правка модалкой перезаписывает их; существующим филиалам график досоздаём.
**Зачем:** п. 8 аудита — сейчас сетка часов рисуется всегда пустой, потому что данных нет и не было.
**Где:** `back/routers/studio/router.py:19–37` (create_branch), `:51–70` (update_branch), `back/schemas/studio/studio.py` (BranchUpdate + WorkingHoursRead уже есть), новая миграция Alembic (бэкфилл).
**Что конкретно сделать:**
1. **Дефолт при создании:** после `db.add(branch)` — 7 строк `BranchWorkingHours`; источник: часы студии из онбординга (`StudioWorkingHours` по `studio_id`), если их нет — пн–пт 09:00–21:00, сб–вс `is_open=False`.
2. **Правка:** `working_hours: list[WorkingHoursWrite] | None` в `BranchUpdate` (day_of_week 0–6, is_open, open_time, close_time). Пришёл список — полная замена семи строк (upsert по `uq_branch_day`); не пришёл — не трогаем.
3. Валидация: day_of_week уникальны и покрывают 0–6, `open_time < close_time` для открытых дней → 422 с текстом.
4. **Бэкфилл:** data-миграция — всем филиалам без строк графика досоздать дефолт (та же логика, что п.1). Прогнать на копии базы.
5. `BranchDetail.working_hours` уже отдаётся — формат ответа не меняется.
**⚠️ Подводные камни:** onboarding-часы могут отсутствовать у старых студий — дефолт пн–пт обязателен как fallback; не забыть `selectinload` при апдейте, чтобы не поймать lazy-load в async.
**Готово, когда:** POST /branches сразу возвращает филиал, у которого GET показывает 7 дней; PATCH с новым графиком меняет сетку; у всех старых филиалов график появился.

### 2. Фронт: секция «Часы работы» в модалке филиала · 🟡 · 1:00
**Простыми словами:** в пересобранную модалку «Редактировать филиал» (V3-3) добавить секцию из семи строк: тумблер «выходной» + время с/до. Компонент готов — `WorkingHoursEditor` из кита (V3-2, задача 5.7).
**Где:** модалка филиала (`Catalog/components/modals/`), `front/src/api/studio/studio.types.ts` (BranchUpdate + working_hours).
**Что конкретно сделать:**
1. Инициализация из `branch.working_hours` (в `BranchDetail` уже приходит).
2. Отправлять `working_hours` в PATCH только если пользователь что-то менял (грязный флаг) — не гонять 7 строк при правке одного названия.
3. Сетка часов в обзоре филиала обновляется сама — инвалидация `['branch', id]` уже есть (V3-1).
4. В визарде создания график **не** спрашиваем — дефолт с сервера (решение аудита).
**Готово, когда:** поменял часы вторника в модалке → сетка в обзоре показывает новые; выходной рисуется как «Закрыто».

## Definition of Done эпика

- У каждого филиала есть график работы (дефолт при создании, редактирование в модалке, бэкфилл старых).
- Фото работают в обе стороны: загрузка (филиал и зал) и показ (шапка с fallback-цепочкой зал → филиал → заливка).
- Кнопки шапки правят выбранную сущность; дублирующих кнопок нет.
- На странице ни одной выдумки: расписание услуги — из занятий, «записей/мес» — честное, категории — ключи с переводом, валюта — студии, «скопировано» — правда.
- Сборка, линт и тесты чистые; STATUS.md обновлён.
