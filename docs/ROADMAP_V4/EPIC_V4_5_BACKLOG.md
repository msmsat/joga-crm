# Эпик V4-5 — Журнал на выбранном языке + последняя честность

Цель эпика простыми словами: Журнал — единственная большая страница вообще без
i18n: ни одного `t()` во всей папке, включая модалки, тосты и даты. Владелец
требует «пусть везде в прямом смысле этого слова будет тот язык, который
выбран». Эпик переводит страницу целиком (namespace `journal`, ru + en) и
добивает последние нечестные элементы: демо-чат AI с захардкоженным ответом и
выдуманные точки событий в мини-календаре.

Источник: аудит 3 в `docs/ROADMAP_V4/ROADMAP.md`, пп. 1, 7, 9 (обсуждено
2026-07-14).

**Важно понимать про старт:**
- Инфраструктура готова: i18next подключён (`front/src/i18n.ts`), namespaces
  `common/staff/profile/catalog` — образец структуры и подключения словаря есть.
- Выполнять **последним** в V4: эпики V4-1…V4-4 переписывают и удаляют строки
  в тех же файлах — переводить их до этого значит переводить дважды. Если
  V4-4 задача 5 (валидация) прошла раньше — забрать её строки в словарь здесь.
- Отдельная ловушка — даты: месяцы и дни недели захардкожены русскими массивами
  (`constants.ts:18-19`), а `RightPanel.tsx:268` форматирует день через
  `toLocaleDateString('ru-RU')` — на en-интерфейсе даты останутся русскими.
- AI-чат журнала (`components/AiChat.tsx`) — имитация: ответ «Пока я работаю в
  демо-режиме 🤖» через `setTimeout`. При этом глобальная панель `AIDrawer` уже
  смонтирована на каждой странице (`DashboardLayout.tsx:590`). Двух чатов быть
  не должно.

**Обозначения сложности:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

## Фаза 1 — Словарь

компонентам.

`resources`); компоненты — вся папка `front/src/pages/dashboard/Journal/`.

### 2. Даты по локали · 🟡 · 1:00
**Простыми словами:** месяцы, дни недели и заголовок дня должны переключаться
вместе с языком. Сейчас всё русское намертво.
**Где:** `constants.ts:18-19` (`MONTH_NAMES`, `DAY_NAMES_SHORT`),
`RightPanel.tsx:268` (`toLocaleDateString('ru-RU')`), потребители:
`Toolbar.tsx:103`, `RightPanel.tsx:48, 278`, `Grid.tsx:109`,
`Journal.tsx` (парсер ручного ввода даты не трогаем — он числовой).
**Что конкретно сделать:**
1. Русские массивы удалить; заменить хелперами на `Intl`:
   `monthName(m, lang)` / `weekdayShort(d, lang)` через
   `toLocaleDateString(i18n.language, { month: 'long' / weekday: 'short' })` —
   нативная платформа вместо словарей (месяцы i18next переводить не надо).
2. Заголовок правой панели («Вторник», «15 июля 2026») — та же локаль;
   капитализация первой буквы остаётся (в en она и так есть).
3. Проверить мини-календарь: сетка начинается с понедельника независимо от
   локали (бизнес-решение, не локальное) — не дать `Intl` сдвинуть неделю.
**Готово, когда:** переключение языка мгновенно меняет все даты страницы;
неделя всегда с понедельника.

### 3. Тосты, ошибки и plural-формы через `t()` · 🟡 · 0:45
**Простыми словами:** сообщения тостов («Занятие перенесено», «Записано: 3
клиента»...) и переводы ошибок API — в словарь, с правильными русскими
множественными формами вместо «клиент(а)».
**Где:** `Journal.tsx` (все `showToast(...)`), undo-тосты из V4-3, обработка
`ApiError` (схема из аудита 2 п. 1).
**Что конкретно сделать:**
1. Все success/undo-тексты → ключи `journal.json`.
2. «Записано: N клиент(а)» → i18next plural (`_one/_few/_many` для ru,
   `_one/_other` для en) — костыль «(а)» умирает.
3. Ошибки: известные статусы без осмысленного `detail` (403, 404, 500, сеть) —
   через `t('common:errors.<код>')` (ключи уже заводились для Каталога в
   V3-2 задаче 2 — переиспользовать, не дублировать); осмысленный `detail`
   с бэка (409 «Клиент уже записан», 400 «Мест не может быть меньше…»)
   показывать как есть.
**Готово, когда:** на en-интерфейсе все тосты английские; «Записано 2 клиента /
5 клиентов» склоняется правильно на обоих языках.

**Что конкретно сделать:**

### 5. Точки мини-календаря — от реальных занятий · 🟡 · 1:00
**Простыми словами:** точки «в этот день есть события» в мини-календаре сейчас
захардкожены (`hasEventDays = [3, 7, 9, 12, …]` — `RightPanel.tsx:27`).
Показывать точки по реальным занятиям месяца.
**Зачем:** п. 7 аудита — выдуманные данные в премиальном продукте недопустимы.
**Где:** `RightPanel.tsx` (MiniCalendar), `useSchedule.ts` / новый лёгкий
эндпоинт (решить по факту, см. ниже), `back/routers/schedule/lessons.py`.
**Что конкретно сделать:**
1. Лёгкий бэк-эндпоинт `GET /schedule/lessons/days?month=YYYY-MM` → массив дат,
   в которых есть неотменённые занятия (`SELECT DISTINCT date(start_time)` по
   студии; для тренера — только свои, тот же скоуп, что в `list_lessons`).
   Гонять полный список занятий месяца ради точек — расточительно.
2. Фронт: квери `['journal-days', 'YYYY-MM']` (staleTime побольше, минут 5),
   рефетч при листании месяца в мини-календаре; инвалидация от мутаций занятий
   (создание/отмена меняют точки).
3. Точки прошлого/следующего месяца в «хвостах» сетки — не рисовать (данных
   нет, и не надо: это соседние месяцы).
**Готово, когда:** точки совпадают с днями, где реально есть занятия; создание
первого занятия на пустой день зажигает точку без F5.
