# Эпик N-5 — Real-time без F5, ErrorBoundary, сквозная приёмка

Цель простыми словами: закрыть модуль production-ready. Всё, что меняется на
странице, видно сразу (оптимистика + инвалидация React Query, никакого F5);
падение любого компонента ловит глобальный ErrorBoundary с аккуратным
fallback вместо белого экрана; весь модуль прогнан сквозными сценариями по
ролям, статус в документации обновлён.

**Важно понимать про старт:**
- WebSockets в проекте нет и не появляется (решение в ROADMAP.md):
  «real-time» = React Query — оптимистичные апдейты, инвалидация ключей,
  стандартный `refetchOnWindowFocus` (подключил бота в другой вкладке —
  вернулся, статус подтянулся). Для внутристраничных действий этого
  достаточно, ТЗ допускает.
- ErrorBoundary в проекте **отсутствует вовсе** (проверено grep) — создаём
  глобальный, это новый компонент UI-кита.
- Query-ключи модуля к этому моменту: `['notification-settings']`,
  `['notification-event-toggles']`, `['notify-integrations']`.

**Обозначения сложности:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

## Фаза 2 — Ошибки

### 2. Глобальный ErrorBoundary в UI-кит · 🟡 · 1:00
**Где:** новый `front/src/components/ui/ErrorBoundary.tsx`,
`front/src/components/ui/index.ts`, `DashboardLayout`
(`front/src/layouts/` или где живёт оболочка с Sidebar/Navbar — найти по
импорту `Sidebar` из `components/ui/index`), `CLAUDE.md`.
**Что конкретно сделать:**
1. Классовый компонент (единственный способ ловить ошибки рендера):
   `static getDerivedStateFromError`, `componentDidCatch` с
   `console.error`; fallback — `Card` кита по центру: иконка (inline SVG,
   не эмодзи), `t('common:errorBoundary.title')` («Что-то пошло не так»),
   текст-подсказка и `Button` «Обновить страницу»
   (`window.location.reload()`). Ключи — в `locales/{ru,en}/common.json`.
2. Экспортировать из `components/ui/index.ts`; в `DashboardLayout` обернуть
   область контента (вокруг `<Outlet/>`/children, **внутри** оболочки — при
   падении страницы меню и навбар остаются живыми и дают уйти на другой
   раздел).
3. Добавить строку `ErrorBoundary` в таблицу UI-кита в `CLAUDE.md` §5
   (правило кита: новый общий компонент → строка в таблице).
**⚠️ Подводные камни:** ErrorBoundary не ловит ошибки в обработчиках событий
и промисах — это зона тостов из `onError` мутаций; granular-обёртки вокруг
отдельных виджетов не плодить (одна, на контент страницы).

### 3. Ревизия: ни одного локального error-стейта · 🟢 · 0:15
**Где:** `front/src/pages/dashboard/Notifications/**`.
**Что конкретно сделать:** `grep -rn "setError\|useState<string | null>"`
по папке страницы — после N-3 совпадений быть не должно; найденное — снести

отправки: это валидация, не ошибка запроса, ей можно жить в `Input error`).
