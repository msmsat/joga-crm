# Velora CRM — Roadmap AI (аудит страницы Velora AI)

Аудит страницы `/dashboard/ai` — обсуждено 2026-07-22.

**Действуют общие решения из V3 (на всё приложение):** TanStack Query как кэш
данных (данные без F5) и единый UI-кит (`front/src/components/ui/`: Toast,
ConfirmModal, Switch, Tooltip, Select). Страница AI написана на CSS Modules —
остаёмся в её стиле (правило §5 CLAUDE.md), Tailwind в неё не примешиваем.

## Точка старта (проверено кодом 2026-07-22)

- **БД готова целиком** — `back/models/ai.py`:
  - `StudioAISettings` — model/language/system_prompt + полные конфиги TG/IG
    (enabled, token, username, tone, max_length, off_hours_only,
    handled_count, avg_rating);
  - `AIChatSession` — studio_id, user_id, title(200), preview(500),
    created_at, updated_at (onupdate);
  - `AIChatMessage` — session_id, role(10), text(Text), created_at.
  Таблицы уже в миграциях. Новых миграций роадмэпу нужно ровно две: дефолт
  модели `gpt-4o` → `velora-3.5` (AI-2) и OAuth-поля Instagram (AI-3).
- **Бэкенд — ноль эндпоинтов**: `back/routers/ai/chat.py` — пустой
  `APIRouter()`, `back/schemas/ai/{chat,settings}.py` — заглушки-комментарии.
- **Контракт фронта уже написан, но никем не используется**:
  `front/src/api/ai/ai.api.ts` — 7 методов (`/ai/sessions` CRUD, messages,
  `/ai/settings`). Гэпы: тип `AISettings` не содержит `*_max_length`,
  `*_off_hours_only` частично и статистику; `sendMessage` типизирован как одно
  сообщение (нужен ответ парой user+assistant).
- **Три независимых мок-движка на фронте**: страница
  (`AI/hooks/useAIChat.ts` + `MOCK_SESSIONS`/`MOCK_AI_RESPONSES`), глобальный
  дровер (`components/AIDrawer/hooks/useDrawerChat.ts` + `MOCK_DRAWER_*`),
  AI-строка шапки (`components/ui/Navbar.tsx` ~56–59 — setTimeout-имитация).
  Общий между ними только `AIDrawerContext` (open/close).
- **`MODEL_OPTIONS` в `AI/constants.ts`** — чужие модели (gpt-4o, claude,
  gemini); `useAIAgent.saveConfig` — фейк (ставит флажок `saved` на 2 сек).
- TanStack Query ^5 установлен и работает на других страницах; i18next с
  файлами `front/src/locales/{ru,en}/*.json` (namespace `ai` отсутствует);
  `useToast`/`ConfirmModal`/`Switch` — в UI-ките.

## Общие решения аудита (приняты 2026-07-22)

1. **Одна серверная история чатов на все поверхности.** Страница AI, дровер и
   AI-строка шапки работают через один хук `front/src/hooks/useAssistant.ts`
   (TanStack Query, ключи `['ai','sessions']` / `['ai','messages',id]`) — один
   кэш, поэтому диалог, начатый в шапке, мгновенно виден в списке страницы без
   F5. Сессии личные: скоуп `studio_id + user_id` — никто (включая владельца)
   не видит чужих диалогов.
2. **LLM-агностичный задел.** Весь вызов ИИ изолирован в
   `back/services/assistant.py` с OpenAI-совместимым контрактом
   (`{LLM_BASE_URL}/v1/chat/completions`, env `LLM_BASE_URL`/`LLM_MODEL`).
   Пока env пуст — сервис возвращает честную заглушку на языке студии.
   Подключение реальной нейросети (Ollama/vLLM, SSE, инструменты — см.
   `docs/BACKLOG/EPIC_6_BACKLOG.md`) не изменит ни одного эндпоинта и ни
   одного байта фронтового контракта.
3. **Модель одна — «Velora AI 3.5»** (внутренний id `velora-3.5`, храним в
   `StudioAISettings.model`). В селекте — единственный активный пункт плюс
   disabled-заглушка «Скоро появятся» (без выдуманных названий моделей).
4. **Системный промпт принадлежит студии.** Дефолт создаётся сервером при
   первом GET настроек и содержит название студии вместо «Velora»
   (`Ты — вежливый ассистент студии «{Studio.name}»…`).
5. **Telegram = токен бота с серверной проверкой `getMe`; Instagram = OAuth
   (Instagram API with Instagram Login, Meta), не токен-инпут.** Тумблеры
   агентов гейтятся и в UI, и на сервере: PATCH `enabled=true` без
   подключённого канала → 400. Согласовано с соседними роадмэпами: канал
   instagram в уведомлениях выключен (ROADMAP_NOTIFICATIONS, решение 2),
   в Онлайн-записи Insta — информационная модалка (ROADMAP_BOOKINGS, блок 1);
   agent-данные живут только в `StudioAISettings`.

## Пункты аудита → что делаем

### 1. Локализация и глобальный UI

**Сейчас:** ни одного `useTranslation` во всех трёх поверхностях; хардкоды
«Сегодня/Вчера/Ранее», `ru-RU` в `toLocaleDateString` (`LeftPanel.tsx:28,39`),
русские тексты в `Navbar.tsx` и `AIDrawer/constants.ts`; ошибок не бывает,
потому что нет запросов; валидации инпутов нет.

**Решение:** namespace `ai.json` (ru+en), все строки через `t()`, даты через
`Intl` с `i18n.language`; все ошибки мутаций/загрузок — глобальный `useToast`;
деструктив (удаление чата, отключение Instagram) — `ConfirmModal` (danger);
строгая клиентская валидация каждого инпута (длины, формат токена) до
отправки. Денежных значений своих у страницы нет — правило валюты сводится к
запрету добавлять `₽`-хардкоды. → **эпик AI-4** (+тосты подключаются по ходу
AI-1…AI-3).

### 2. Ядро чатов: реальная история

**Сейчас:** ответ — случайная фраза из четырёх, история — `MOCK_SESSIONS`,
«загрузка сессии» подставляет два выдуманных сообщения; `/ai` на сервере пуст.

**Решение:** эндпоинты сессий/сообщений (список с title, preview последнего
сообщения, message_count, датами — строго из БД), `useAssistant` на TanStack
Query с оптимистичной отправкой, выпил всех `MOCK_*`, интеграция на трёх
поверхностях: страница, дровер, AI-строка. → **эпик AI-1**.

### 3. Настройки AI и промпты

**Сейчас:** селекты модели/языка — локальный стейт, «Сохранить» — фейковый
флажок; в списке моделей gpt-4o/claude/gemini; промпт — константа со словом
«Velora».

**Решение:** GET/PATCH `/ai/settings` (get-or-create; при создании промпт
рендерится с названием студии), миграция дефолта модели, селект «Velora AI
3.5» + заглушка «Скоро появятся», персист языка/модели/промпта в БД.
→ **эпик AI-2**.

### 4. AI-агенты: Telegram и Instagram

**Сейчас:** оба канала — одинаковые токен-инпуты, ничего не сохраняется,
тумблеры свободно щёлкаются, статистика — нули из локального дефолта.

**Решение:** Telegram — серверная проверка токена через `getMe` (сохраняет
token+username), тумблер заблокирован без токена, инструкция BotFather;
Instagram — полный OAuth-флоу Meta (oauth-url → callback → long-lived token
на 60 дней + refresh), инструкция про профессиональный аккаунт, disconnect
через ConfirmModal. Статистика — реальные `*_handled_count`/`*_avg_rating`
из БД (честные нули до подключения LLM). → **эпик AI-3**.

### 5. Задел на будущее (реальная нейросеть)

Закрывается решением 2: контракты эндпоинтов и payload'ы спроектированы как
для живого LLM; включение = поднять сервер модели и заполнить `LLM_BASE_URL`.
Вне рамок этого роадмэпа: SSE-стриминг, инструменты ассистента, вебхуки
автоответчиков TG/IG Direct — всё это требует живой модели и описано в
`docs/BACKLOG/EPIC_6_BACKLOG.md` (фазы 1–3).

Порядок и зависимости: **AI-1 первым** — эндпоинты чатов и `useAssistant`
это фундамент, на который AI-2 добавляет настройки (тот же get-or-create

ассистента отвечает на языке из настроек — появляется смысл селекта).
**AI-3 третьим** (PATCH `/ai/settings` из AI-2 — транспорт для tg/ig-полей).
**AI-4 последним** — переводит и полирует уже реальный UI, а не моки.
Внутри каждого эпика порядок строгий: БД → бэкенд → фронт-API → стейт → UI.
