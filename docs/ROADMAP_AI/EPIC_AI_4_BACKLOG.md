Цель эпика простыми словами: три AI-поверхности начинают уважать язык
пользователя (ни одной русской строки в коде и ни одного `ru-RU`-хардкода),

не миганием пустоты, и весь роадмэп закрывается чистым build/lint-прогоном.

Источник: аудит в `docs/ROADMAP_AI/ROADMAP.md`, п. 1. Эпик последний: он
переводит и полирует уже реальный UI (AI-1…AI-3), а не моки.

**Важно понимать про старт:**
- i18next настроен, namespaces лежат в `front/src/locales/{ru,en}/*.json`
  (образец подключения — `loyalty.json` из V5-2); namespace `ai` отсутствует.
- Хардкоды локали: `LeftPanel.tsx:28` («Сегодня/Вчера/Ранее»),
  `LeftPanel.tsx:37-39` (`мин`/`ч`, `toLocaleDateString('ru-RU')`); все
  тексты `Navbar.tsx` (плейсхолдер AI-строки, «Velora AI думает над
  ответом»), `AIDrawer` и модалки агентов — по-русски в коде.
- Денежных значений у страницы нет — правило валюты здесь сводится к запрету
  добавлять `₽`-хардкоды (суммы в ответах ассистента — текст от бэка).
- Тумблер светлой/тёмной темы и i18n остальных страниц — не сюда
  (см. ROADMAP, «Вне рамок»).

**Обозначения сложности:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

## Фаза 1 — Язык

### 1. `ai.json` (ru+en) + `t()` по трём поверхностям · 🟡 · 1:30

**Простыми словами:** все строки AI-интерфейсов — в словарь; интерфейс живёт
на языке, выбранном пользователем.
**Где:** новые `front/src/locales/ru/ai.json` и `front/src/locales/en/ai.json`;
регистрация namespace в `front/src/i18n.ts` (по образцу `loyalty`); вся папка
`front/src/pages/dashboard/AI/`, `front/src/components/AIDrawer/`, AI-части
`front/src/components/ui/Navbar.tsx`.
**Что конкретно сделать:**
1. Собрать в `ai.json` ВСЕ строки: заголовки панелей («История чатов»),
   кнопки («Новый чат», «Сохранить», «Проверить и подключить», «Подключить
   Instagram», «Отключить», «Отвязать»), плейсхолдеры («Спросите AI или
   поставьте задачу…», поле сообщения), статусы («Velora AI думает над
   ответом», «Скоро появятся»), инструкции BotFather и Instagram (массивами
   шагов), тексты empty-state, тултипы гейтов, тексты ConfirmModal, подписи
   статистики, тексты тостов успеха.
2. Тексты ошибок — по общему правилу: `detail` бэка (`tg_token_required`,
   `ig_not_connected`, `invalid_bot_token`, `assistant_unavailable`) →
   ключи `ai:errors.*`; сетевые/статусные — переиспользовать `common:errors.*`,
   если ключи уже заведены (не плодить третий словарь ошибок).
3. `LANGUAGE_OPTIONS`/`MODEL_OPTIONS` в `constants.ts`: `label` — через `t()`
   в месте рендера (константы держат только ключи) — по решению V5-2
   «значения — ключи, отображение — locales».
4. en-словарь — полный перевод, не копия русского.
**⚠️ Подводные камни:** язык интерфейса CRM (i18next) и язык ответов
ассистента (`AISettings.language`, селект LeftPanel) — РАЗНЫЕ настройки; не
склеивать. Плейсхолдер AI-строки в шапке живёт в `Navbar.tsx` — это общий
компонент кита, переводить через тот же namespace `ai`.
**Готово, когда:** `grep -r "[А-Яа-я]" front/src/pages/dashboard/AI
front/src/components/AIDrawer` находит только `ru/ai.json`-ключи в тестах —
т.е. ноль кириллицы в `.tsx/.ts`; переключение языка меняет все три
поверхности без F5.

### 2. Даты и «N мин назад» через `Intl` · 🟢 · 0:30

**Простыми словами:** «Сегодня/Вчера/5 мин/22 июл» и время сообщений — на
языке пользователя, без `ru-RU` в коде.
**Где:** `LeftPanel.tsx` (`groupByDay`, `timeAgo`), `AIDrawer/components/
HistoryView.tsx`/`MessageBubble.tsx`, `AI/components/MessageBubble.tsx`.
**Что конкретно сделать:**
1. `timeAgo` → `Intl.RelativeTimeFormat(i18n.language, { numeric: 'auto' })`
   для минут/часов; старше суток —
   `Intl.DateTimeFormat(i18n.language, { day: 'numeric', month: 'short' })`.
2. Ярлыки групп «Сегодня/Вчера/Ранее» → `t('ai:history.today'|'yesterday'|'earlier')`.
3. Время сообщения —
   `Intl.DateTimeFormat(i18n.language, { hour: '2-digit', minute: '2-digit' })`.
4. Хелпер один на три поверхности — положить в
   `front/src/pages/dashboard/AI/utils.ts`? Нет: используется и дровером →
   `front/src/lib/datetime.ts` (новый общий модуль; если в проекте уже есть
   аналогичный хелпер дат — переиспользовать его, не создавать второй).
**Готово, когда:** grep `ru-RU` по AI-папкам пуст; en-интерфейс показывает
«5 min ago / Jul 22».

## Фаза 2 — Полировка

### 3. Скелетоны и empty-state вместо мигания · 🟢 · 0:45

**Простыми словами:** пока история грузится — скелетон списка; чатов нет —
приветственный empty-state; ошибка загрузки — тост и кнопка «Повторить», а
не вечная пустота.
**Где:** `LeftPanel.tsx`, `ChatPanel.tsx`, `AIDrawer/components/HistoryView.tsx`
(+ `EmptyStateSVG.tsx` уже нарисован — переиспользовать);
стили — в существующих `.module.css` рядом.
**Что конкретно сделать:**
1. `sessionsLoading` → 4 скелетон-строки списка (шиммер, как на других
   страницах); `messagesLoading` при открытии сессии → скелетон двух пузырей.
2. Пустая история (`sessions.length === 0`, загрузка завершена) —
   empty-state с `EmptyStateSVG` и текстом `t('ai:empty.title')`.
3. `isError` запросов → error-тост один раз + инлайн-кнопка «Повторить»
   (`refetch`); молча показывать пустоту запрещено (правило «ошибки видимы
   всегда»).
4. Query-кэш уже даёт мгновенный повторный заход (staleTime 30 с) — ничего
   дополнительно не изобретать.
**Готово, когда:** первый заход — скелетон → данные без прыжков раскладки;
обрыв сети — тост + «Повторить», после восстановления список оживает.

### 4. Ревизия валидации всех инпутов + финальный прогон · 🟡 · 0:45

**Простыми словами:** пройти каждый инпут трёх поверхностей чек-листом
«пусто / максимум / мусор», убедиться, что на сервер не уходит невалидное,
и закрыть роадмэп чистым прогоном.
**Где:** все инпуты AI-поверхностей; `docs/TZ/STATUS.md` (раздел Velora AI).
**Что конкретно сделать:**
1. Чек-лист по инпутам (клиентская проверка + серверное зеркало уже из
   AI-1…AI-3):
   - поле сообщения (страница/дровер/шапка): trim-пусто — не отправлять,
     4000 макс (`maxLength` на элементе + счётчик при >3500), Enter при
     `isThinking`/`isPending` — игнор;
   - системный промпт: 2000 + счётчик (AI-2);
   - Telegram-токен: формат-паттерн до кнопки (AI-3);
   - селекты модель/язык/тон: свободного ввода нет — валидны по построению;
   - `tg_max_length`/`ig_max_length` (если редактируются в модалке):
     число 50–4000, `Input error`-состояние + подсказка.
2. Прогон: `cd front && npm run build && npm run lint` — ноль ошибок;
   бэк: `pytest back/tests` зелёный.
3. Ручной smoke всего роадмэпа: вопрос из шапки → сессия на странице без
   F5 → ответ-заглушка на языке настроек → смена модели невозможна мимо
   `velora-3.5` → TG-токен подключается и гейтит тумблер → IG проходит OAuth
   на тестовом приложении → все ошибки видны тостами.
4. Обновить `docs/TZ/STATUS.md`: раздел «Velora AI» из «красивая имитация» →
   фактическое состояние (реальная история/настройки/агенты; LLM — заглушка
   за `LLM_BASE_URL`).
**Готово, когда:** чек-лист пройден, прогоны зелёные, STATUS.md отражает
реальность.
