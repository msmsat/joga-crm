# Целевая аудитория

Приложение доступно только после регистрации — гостевого режима нет. Текущий фокус — студии **пилатеса и йоги** (другие вертикали добавятся позже).

В MVP действует простая модель доступа: **три фиксированные роли, без конструктора прав**. У каждого сотрудника два независимых поля: **должность** (свободный текст — «Тренер по пилатесу», «Бухгалтер» — чисто подпись, на доступ не влияет) и **роль доступа** (селект). Матрица галочек RolePermission в MVP скрыта и не проверяется — кастомные роли появятся позже как фича тарифа Business.

1. **Владелец (owner)** — тот, кто создал студию; не назначается через форму. Видит и может всё.
2. **Администратор** — ресепшн/управляющий: журнал, записи, клиенты, продажи. Финансы, настройки и Billing закрыты.
3. **Тренер** — своё расписание и свои клиенты, отмечает посещения. Чужие расписания и финансы закрыты.
4. **Клиент студии** — в CRM не заходит, живёт в брендированном мини-приложении (TG, Insta, web): запись, абонемент, баллы, история.

При добавлении сотрудника владелец выбирает доступ из двух вариантов (Полный(как администратор)/ Ограниченый(как тренеру)) — по одной строке пояснения, без настроек.
