# Стек технологий

## Стек технологий (Нефункциональные требования)

**Фронтенд:** React (TypeScript), Vite, React Router, Recharts (графики), CSS Modules, TanStack Query (серверный кэш: данные без F5, мгновенная отдача из кэша). Фронтенд и UI уже написаны, дизайн не трогать.

**Кэш данных (TanStack Query) — паттерн для новых страниц и будущих аудитов:**
- Ключ = сущность (+ id при детали); все ключи — в `front/src/api/queryKeys.ts`, не строки в хуках.
- Любая мутация в `onSuccess` инвалидирует **все** ключи, где изменённая сущность видна (напр. CRUD зала трогает и `['branch', id]`, и `['branches']` — счётчик залов в списке).
- Трансформация под UI — через `select` квери, не отдельным ключом и не в компоненте.
- Не плодить `useState`+`useEffect`+ручной `refetch` — это то, что Query заменяет.
- **Мигрировано:** Каталог (`['branches']`, `['branch', id]`, `['services']`). **Очередь (по мере аудитов):** Клиенты, Журнал, Сотрудники, Финансы, Отчёты.

**Бэкенд и база данных:** Python, FastAPI, PostgreSQL (через SQLAlchemy + asyncpg), Alembic (миграции базы данных), Pydantic (валидация данных), JWT (авторизация).
