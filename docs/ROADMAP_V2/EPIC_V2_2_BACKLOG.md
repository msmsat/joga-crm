# Эпик V2-2 — Единый UI-кит: тосты, подтверждения, списки

Цель эпика простыми словами: одинаковые вещи в приложении должны выглядеть и вести себя одинаково — и существовать **в одном экземпляре**. Сейчас тостов четыре реализации, модалка «удалить? — подтвердить» на каждой странице своя, выпадающие списки (язык, время, фильтры) свёрстаны заново в каждом месте. Сводим к трём общим компонентам и раскатываем по всем страницам.

Порядок из аудита 1 (`ROADMAP.md`, п.8): тосты → модалки подтверждения → выпадающие списки.

**Важно понимать про старт:**
- Общий `Toast` + `ToastProvider` уже созданы в эпике V2-1 и работают в Сотрудниках. Здесь — только миграция остальных страниц и удаление дублей.
- Это эпик-«уборка»: новой функциональности нет, поведение страниц не меняется (рефакторинг = реорганизация). Единственное новое — ошибки API начинают всплывать и там, где раньше молчали.
- Дизайн-эталон каждого компонента — лучший из уже существующих вариантов (премиум-стиль по CLAUDE.md §6), а не новый дизайн.

**Обозначения сложности:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

### 5. Ошибки API — тост по умолчанию везде · 🟡 · 1:30
**Простыми словами:** в эпике V2-1 ошибки сервера стали видимы в Сотрудниках. Теперь то же правило — для всего приложения: любой необработанный неуспех запроса (403, 409, 422, 500, сеть) показывает error-тост с текстом от сервера. Страница может перехватить ошибку и показать что-то умнее — но «молча проглотить» больше невозможно по умолчанию.
**Где:** `front/src/api/client.ts` (нормализация ошибок уже сделана в V2-1, задача 7).
**Что конкретно сделать:**
1. В клиенте — обработчик по умолчанию: если вызывающий код не пометил запрос как «сам обработаю», ошибка уходит в error-тост.
2. 402 (пейволл) продолжает открывать глобальную модалку апселла — не тост.
3. Пройтись по местам, где сейчас `catch {}` глотает ошибку — убрать заглушки, положиться на общий механизм.
**⚠️ Подводные камни:** не задвоить сообщения там, где страница уже показывает свой error-тост.
**Готово, когда:** выключенный бэкенд превращает любое действие в понятный тост, а не в тишину.
