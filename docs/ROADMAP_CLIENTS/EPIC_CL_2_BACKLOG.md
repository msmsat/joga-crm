### 2.1 Список/категории/профиль на TanStack Query · 🔴 · 2:30
**Простыми словами:** `useClientsList` вручную клеит страницы пагинации, категории
грузятся отдельным `useEffect`, профиль — ручным `getProfile → patchLocal`. Переводим
загрузку на TanStack Query — тот же паттерн, что уже работает в Каталоге
(`Catalog/hooks/useCatalogList.ts`), кэш и фоновое обновление достаются бесплатно.
**Где:** `front/src/api/queryKeys.ts`, `front/src/pages/dashboard/Clients/hooks/useClientsList.ts`, `Clients.tsx`.
**Что конкретно сделать:**
1. В `queryKeys.ts` добавить: `clients(search, category)`, `client(id)`, `clientCategories`,
   `clientEvents(id, filter)`, `clientActivity(id)`, `clientNotes(id)`, `studioSettings`.
2. `useClientsList` → `useInfiniteQuery` (offset = pageParam, «Загрузить ещё» = `fetchNextPage`,
   `hasMore` — из `total`). Дебаунс поиска оставить как есть.
3. Категории — `useQuery(clientCategories)` вместо ручного `loadCategories` (`Clients.tsx:34`).
4. Профиль — `useQuery(client(id), enabled: панель открыта)` вместо ручного
   `getProfile → patchLocal` (`Clients.tsx:58–63`). Повторное открытие клиента — мгновенно из кэша.
5. `patchLocal` / `removeLocal` / `reload` удалить.
**Готово, когда:** страница работает как раньше, но ни одного ручного «подгона» списка
в коде не осталось; `npm run build` чистый.

### 2.2 Все мутации через useMutation + инвалидация · 🟡 · 2:00
**Простыми словами:** каждое действие (создать, удалить, заморозить, сменить статус, тег,
заметка, бонус, запись) должно после успеха инвалидировать затронутые ключи — и всё на
странице (список, счётчики категорий, профиль, события) обновляется само, без F5.
**Где:** `useClientActions.ts`, `Clients.tsx`, `AddClientModal.tsx`.
**Что конкретно сделать:**
1. Каждый вызов `clientsApi.*` завернуть в `useMutation`; `onSuccess` → инвалидация:
   всегда `clients` + `client(id)`; для create/delete/freeze/status — ещё `clientCategories`;
   для booking/bonus/freeze — `clientEvents(id)`; для заметок — `clientNotes(id)`.
2. Оптимистичные локальные копии (`localTags`, `notes`, `bonusPoints` в `useClientActions`)
   убрать — мгновенность даёт кэш, правду — инвалидация.
3. Костыль `handleFreezeChange`/`onFreezeChange` (`Clients.tsx:88–92`) умирает.
4. Ошибки любой мутации → `toast.error(errorMessage(err, t))` (см. 1.4 в CL-1),
   успехи — success-тост.
**Готово, когда:** после любого действия все виджеты страницы (счётчики табов, карточка
в списке, панель) актуальны без F5 и без ручного кода синхронизации.

### 2.3 Кликабельные контакты · 🟢 · 0:45
**Простыми словами:** блок «Контакты» сейчас только копирует значение в буфер. Телефон и
email должны быть настоящими ссылками, плюс строка Telegram.
**Где:** `ClientProfileSlider.tsx:532–554` (блок контактов).
**Что конкретно сделать:**
1. Телефон → `<a href="tel:+7…">`, email → `<a href="mailto:…">`.
2. Новая строка Telegram → `https://t.me/+<цифры телефона>` (открывает чат, если номер
   зарегистрирован в TG). WhatsApp уже открывается через `wa.me` в панели «Сообщение» — не трогаем.
3. Копирование не терять: отдельная маленькая иконка «копировать» справа в строке.
**Готово, когда:** клик по телефону открывает звонилку, по email — почтовый клиент,
по Telegram — чат.
