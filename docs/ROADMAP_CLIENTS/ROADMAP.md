## Важно понимать про старт — бэкенд почти готов, не переписывать

- Список, поиск, категории со счётчиками, пагинация — реальные, серверные
  (`back/routers/clients/profiles.py`).
- Уже есть API: профиль, заметки CRUD (включая delete и полный список), теги, статус,
  заморозка, дата регистрации, удаление, запись на занятие (с проверкой мест и дублей),
  бонус (пишет `LoyaltyPointTransaction`), события (визиты/оплаты/заморозки абонемента),
  **активность по месяцам за год** — `GET /clients/{id}/activity` (`profiles.py:385`) —
  фронт его просто не зовёт.
- Схема `ClientUpdate` уже написана и даже импортирована в роутер (`profiles.py:33`) —
  эндпоинта под неё нет. Мёртвый импорт = готовое ТЗ.
- Быстрая запись уже грузит **реальные занятия** дня (`useClientActions.ts:134`) —
  не хватает только листания дальше 7 дней.
- Инфраструктура с прошлых аудитов готова: TanStack Query (`front/src/api/queryClient.ts`,
  `queryKeys.ts` — там прямо написано «очередь миграции: Клиенты»), `errorMessage.ts`,
  общий Toast (`front/src/components/ui/Toast.tsx`), общий `ConfirmModal.tsx`,
  хелпер валюты `getCurrencySymbol` (`front/src/components/UI.tsx:482`).
- Телеграм: у клиента в БД уже есть `tg_id` (`back/models/client.py:14`), `TG_BOT_TOKEN`
  описан в `.env.example`, `aiogram` уже в `requirements.txt`. Честное ограничение:
  Bot API не умеет писать по номеру телефона — только по `chat_id` (детали в CL-5).
- У `Reservation` уже есть `created_at` и `cancelled_at` (`back/models/schedule.py:67,69`) —
  события «записался»/«отменил» выводимы без новых таблиц.
- Абонементы для модалки добавления: эндпоинт списка пакетов уже есть
  (`GET /loyalty/subscriptions`, `back/routers/loyalty/packages.py:51`), фронт-обёртка
  тоже (`loyaltyApi.getSubscriptionPackages()`); образец «продать абонемент + деньги
  в кассу + баллы» — `sell_subscription` (`back/routers/clients/subscriptions.py:39`).

**Обозначения сложности в бэклогах:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

## Финальная задача — Сквозной прогон и фиксация в STATUS · 🟡 · 1:00

Делается после всех эпиков.

**Что конкретно сделать:**
1. Чек-лист: создать клиента → счётчики табов обновились; модалка не пускает дальше
   без телефона/email/города; клиент с оплаченным абонементом → абонемент активен,
   операция в Финансах, баланс кассы вырос, баллы начислены; inline-правка телефона видна
   в списке; тег/заметка добавляются и удаляются; запись на занятие через 2 недели →
   событие «записался» в ленте; бонус → баланс + событие (+ Телеграм при `tg_id`);
   заморозка → бейдж и событие; удаление — только через подтверждение; график = реальные
   визиты; en-локаль без русских строк; валюта — из настроек студии; переключение
   клиентов — без дерганья панели.
2. `npm run build && npm run lint` — чисто.
3. Обновить `docs/TZ/STATUS.md` (раздел Клиенты) и проставить статусы эпиков в этом файле.

**Готово, когда:** чек-лист пройден, статус зафиксирован.

## Definition of Done роадмапа

- Ни одного выдуманного числа и ни одной русской строки-хардкода на странице; валюта — студии.
- Любое действие обновляет страницу через инвалидацию кэша — ручных `patchLocal`/`reload` нет.
- Контакты кликабельны; поля профиля редактируются на месте; заметки и теги — полный CRUD.
- Лента событий показывает записи, отмены, оплаты, заморозки и бонусы из реальных таблиц.
- Удаление — только с подтверждением; бонус уходит клиенту в Telegram при наличии `tg_id`.
- Общие компоненты (Toast, ConfirmModal, TanStack Query) переиспользованы, ничего
  app-wide не построено; сборка и линт чистые.
