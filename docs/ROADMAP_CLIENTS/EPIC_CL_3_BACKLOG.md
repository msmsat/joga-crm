### 3.1 Бэк: PATCH /clients/{id} на готовой схеме · 🟢 · 0:45
**Простыми словами:** схема `ClientUpdate` (имя, фамилия, телефон, email, дата рождения,
город, источник) написана (`back/schemas/clients/clients.py:19`) и импортирована в роутер
(`profiles.py:33`), но эндпоинта нет — импорт мёртвый.
**Где:** `back/routers/clients/profiles.py` (рядом с `update_client_status`),
`front/src/api/clients/clients.api.ts`.
**Что конкретно сделать:**
1. `@router.patch("/{client_id}")`: `_get_client_or_404` → `body.model_dump(exclude_unset=True)`
   → `setattr` по полям → commit → `OkOut`.
2. Фронт: `clientsApi.update(id, payload)` (`client.patch('/clients/{id}', …)`), тип
   `ClientUpdate` в `clients.types.ts`.
**Готово, когда:** PATCH с одним полем меняет только его; чужой client_id → 404.

### 3.2 Inline-редактирование полей в панели · 🟡 · 2:00
**Простыми словами:** дата регистрации уже редактируется по клику прямо в шапке панели
(`ClientProfileSlider.tsx:444–461`) — распространяем этот же паттерн на телефон, email,
день рождения и город.
**Что конкретно сделать:**
1. Обобщить существующий inline-edit в мини-компонент `InlineEdit` (локально в
   `Clients/components/`), день рождения — нативный `<input type="date">`.
2. Клик по значению в блоке «Контакты» → поле ввода → blur/Enter → мутация PATCH (3.1)
   → инвалидация `client(id)` + `clients`. Esc — отмена.
3. Совместить с кликабельностью (2.3): ссылка — по значению, карандаш-иконка — редактирование.
**Готово, когда:** все 4 поля редактируются на месте, изменения видны в списке без F5,
ошибка сервера — error-тост.

### 3.3 Заметки: кнопка удаления + полный список · 🟢 · 1:00
**Простыми словами:** API заметок готов целиком (create/update/**delete**/list), но в UI
удаления нет (`useClientActions` не дёргает `clientsApi.deleteNote`), а профиль отдаёт
только 3 последних заметки (`profiles.py:286` — срез `[:3]`).
**Что конкретно сделать:**
1. Вкладка «Заметки» грузит полный список: `useQuery(clientNotes(id))` → `GET /clients/{id}/notes`.
2. Кнопка удаления у заметки → общий `ConfirmModal` → мутация delete → инвалидация.
3. Создание/правка — уже работают; перевести на мутации (2.2).
**Готово, когда:** заметок видно сколько есть; создать/изменить/удалить работают без F5.
