# EPIC R11 — Redesign: списки, модалки и тепловая карта

**Цель:** три места, где Отчёты выглядят как черновик, доводятся до уровня
остального продукта: список клиентов из сегмента, модалка тренера и модалка
тайм-слота. Плюс тепловая карта перестаёт быть сеткой квадратов и становится
плотной временной шкалой.

**Аудит:** пункты 14, 15, 16 (UI), 17.
**Зависимости:** R8 (`EmptyState`), R9 (подсказки в порталах). **Оценка: ~8:30.**

## Backend Architecture

Новых эндпоинтов не нужно — все данные уже приходят:

| Экран | Источник | DTO |
|---|---|---|
| Список клиентов | `GET /analytics/clients-report/segment`, `…/week` | `SegmentClientRow{ id, name, last_name, phone, last_visit_date, value }` |
| Модалка тренера | `GET /analytics/team/{id}` | `TrainerDetailRead{ load_by_weekday[], revenue_series[], top_lessons[], returned_clients, total_clients, return_rate_pct }` |
| Модалка слота | `GET /analytics/utilization/slot?weekday&hour` | `SlotLessonRow{ id, date, name, teacher_name, hall, occupied, total_spots, status }` |

Единственная возможная добавка — если при верстке карточки клиента не хватит
поля: сначала проверить `SegmentClientRow`, и только если поля правда нет —
дописать его в `back/schemas/analytics/reports.py` + `retention.py`. Новых
эндпоинтов ради этого не заводить.

## Frontend Architecture

### Задача 1. `Reports/components/shared/ClientListModal.tsx` (новый)

Заменяет `DrilldownModal` в `ClientsTab.tsx` (использования на строках 203–214).
`DrilldownModal` остаётся для операций и занятий — его не трогаем.

```tsx
export interface ClientListModalProps {
  open: boolean;
  onClose: () => void;
  title: string;                       // название сегмента
  subtitle?: string;                   // описание сегмента из i18n
  rows: SegmentClientRow[];
  valueLabel: string;                  // «Визитов» / «Потрачено» / «Дней без визита»
  loading?: boolean;
  onCampaign?: () => void;             // ведёт в Лояльность с предзаполненным сегментом
}
```

Строение (`ModalShell size="lg"`, левая колонка — контекст сегмента):

- **Левая панель (`left`)**: крупное число клиентов (32px/900), название
  сегмента, описание, инлайн-SVG иллюстрация сегмента, внизу — `PrimaryButton`
  «Запустить кампанию» (если передан `onCampaign`) и `GhostButton` «Скачать CSV».
- **Шапка контента**: `Input` с иконкой поиска — фильтрация по имени и телефону
  на клиенте (`useMemo`, без запроса); справа — счётчик «Показано N из M».
- **Строка клиента** — карточка, не строка таблицы:
  - аватар-кружок 36px с инициалами (`getInitials(name, last_name)` — хелпер
    уже есть в проекте, см. Клиентов) на персиковой подложке
    `rgba(249,160,139,0.14)`, текст `#C07060`, 13px/800;
  - имя 13.5px/700 `var(--text)`, под ним телефон 12px `var(--text3)`;
  - справа — `value` крупно (15px/800) + `valueLabel` мелко, ниже — последний
    визит человеческим текстом: «сегодня», «3 дня назад», «21.07» (хелпер
    `fmtRelativeDate` в `front/src/lib/format.ts`, новый);
  - hover: фон `rgba(249,160,139,0.06)`, радиус 12, сдвиг стрелки-шеврона на 2px;
  - клик → `/dashboard/clients?client={id}` (поведение сохраняется).
- **Анимация**: `framer-motion`, `motion.div` на каждой строке с
  `initial={{ opacity: 0, y: 6 }}`, `animate`, `transition={{ delay: Math.min(i * 0.02, 0.3) }}`
  — каскад входа, но с потолком задержки, иначе на 200 клиентах хвост въезжает
  секундами.
- **Скролл**: контейнер списка `maxHeight: 'min(60vh, 520px)'; overflowY: 'auto'`
  с классом `ms-scroll` (тонкий персиковый скроллбар уже в `ModalShell`).
- **Пусто**: `<EmptyState size="sm" icon="clients">` из R8 (до модалки дело
  обычно не доходит — пустой сегмент отсекается Toast'ом, но список может
  опустеть после поиска: тогда «Никого не нашлось по запросу «…»»).

Экспорт CSV: перенести туда логику из `ClientsTab.tsx:115–129`
(`exportDrilldownList`) — в компоненте ей место, во вкладке она мусор.

**Ограничение (осознанное):** без виртуализации. До ~500 строк каскад и скролл
держатся; если сегменты начнут отдавать тысячи — добавлять пагинацию на бэке,
а не виртуальный список на фронте.

### Задача 4. `schedule/Heatmap.tsx` — плотная шкала (аудит 16)

- Ячейка: `aspectRatio: 1.4` → `height: '14px'` (было ~70px — ровно ÷5),
  `borderRadius: '4px'`, `gap: '3px'`.
- Текст `%` из ячейки убирается (в 14px он не читается) — процент остаётся в
  `Tooltip` кита, который в R9 научился не уезжать за край экрана.
- Подписи часов — каждая вторая строка (`hour % 2 === 0`), 10px/600, чтобы
  колонка времени не рябила.
- Часы — как сейчас, от самого раннего занятия до самого позднего
  (`Heatmap.tsx:33–46`); фолбэк при пустых данных 9:00–20:00 сохраняется, но
  вместо пустой сетки показывается `EmptyState size="sm" icon="calendar"`.
- Пустые слоты: `1px dashed var(--border)` слишком шумно на 14px — заменить на
  заливку `rgba(26,26,26,0.035)` без рамки.
- Hover: `transform: scale(1.06)` на плоской ячейке выглядит как дрожь —
  заменить на `filter: brightness(1.06)` + `outline: 1px solid rgba(249,160,139,0.6)`.
- Легенда остаётся, отступ сверху сокращается до 14px.

## Styling & UI/UX

- Всё строится из кита: `ModalShell/ModalHeader/ModalBody/ModalFooter`, `Card`,
  `Button`, `Input`, `Tooltip`, `InfoHint`, `EmptyState`, `ProgressBar`. Своих
  кнопок и модалок не пишем (CLAUDE.md §5) — не хватает варианта, расширяем кит.
- Акцент персиковый `#F9A08B`/`#FCAE91`; синий `#4A80C4` — только вторая серия
  на графиках; фисташка `#A3C9A8` — успех; пыльная роза `#D88C9A` — негатив.
- Радиусы 12–16, внутренние отступы 20–28, разделители — воздухом, а не
  линиями (в списках допустима линия `1px solid var(--border)` только между
  строками таблицы, которую мы как раз убираем).
- Иконки — inline SVG, эмодзи запрещены.
- Анимации — `framer-motion`: вход 0.28–0.32s,
  `cubic-bezier(0.34, 1.1, 0.64, 1)`; stagger не длиннее 0.3s суммарно;
  уважать `prefers-reduced-motion` (обёртка `useReducedMotion()` из
  framer-motion — при `true` задержки в 0).
- Тёмная тема: только токены (`var(--text)`, `var(--text3)`, `var(--bg-card)`,
  `var(--border)`), hex — исключительно акцентные из ДС.

## Edge Cases

| Ситуация | Поведение |
|---|---|
| Клиент без телефона / без визитов | «—» вместо телефона, «Ни разу не был» вместо даты; строка не ломается |
| Очень длинное имя / название занятия | `text-overflow: ellipsis` + `Tooltip` с полным текстом |
| 300+ клиентов в сегменте | Каскад с потолком задержки 0.3s, скролл внутри модалки; страница под модалкой не скроллится |
| Поиск не дал совпадений | `EmptyState size="sm" icon="search"` с текстом запроса, кнопка «Сбросить» |
| У тренера нет занятий в периоде | Графики — `EmptyState size="sm"`, KPI-карточки нули, модалка открывается штатно |
| У тренера нет рейтинга (`null`) | Блок звёзд не рендерится вовсе (не «0 звёзд» — это ложь) |
| Слот с одним отменённым занятием | Карточка со статусом «отменено», сводка показывает 0 заполненности честно |
| Тепловая карта, 18 часов × 7 дней | Высота полотна ≈ 18 × 17px ≈ 306px — влезает без внутреннего скролла; `overflowX: auto` остаётся для узких экранов |
| `prefers-reduced-motion: reduce` | Все stagger и счётчики выключаются, элементы появляются сразу |

## Definition of Done

- Список клиентов, модалка тренера и модалка слота собраны из кита, с
  иллюстрациями, анимациями и осмысленными пустыми состояниями.
- Тепловая карта помещается на экран без вертикального скролла карточки,
  ячейки — плоские полоски, процент читается в подсказке.
- В новых компонентах нет ни одной своей кнопки/модалки/инпута и ни одного
  эмодзи; все строки через `t('reports:…')` в ru и en.
- Ни один файл вкладок не вырос выше 300 строк (иначе делим на `components/`).
- `npm run build && npm run lint` зелёные.
