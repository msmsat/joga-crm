## Frontend Architecture

### Задача 1. `front/src/components/ui/Card.tsx`

```tsx
style={{
  position: 'relative',
  zIndex: lifted ? 3 : 1,                      // приподнятая карточка всегда сверху
  boxShadow: lifted ? '0 16px 40px -8px rgba(26,26,26,0.1)' : '0 8px 24px -4px rgba(26,26,26,0.04)',
  transform: lifted ? 'translateY(-2px)' : 'translateY(0)',   // не 'none' — иначе прыжок без анимации
  transition: 'transform 0.25s cubic-bezier(0.34,1.2,0.64,1), box-shadow 0.25s ease',
  willChange: 'transform',
  ...style,
}}
```

Три правки, каждая закрывает свой симптом:
- `position/zIndex` — соседняя карточка больше не наезжает на приподнятую;
- `transition` на два конкретных свойства вместо `all` — при ререндере (а он
  происходит на каждый ответ Query) не анимируются padding/цвет/размеры, из-за
  чего интерфейс и «лагал»;
- `translateY(0)` вместо `none` — переход в обе стороны плавный.

**Проверить, что `style`-проп потребителей не перетирает `zIndex`:** в
`ProductsTable.tsx:54` карточка получает `style={{ overflow: 'hidden' }}` —
конфликта нет, но спред `...style` должен остаться последним.

Поповеры внутри карточек (`InfoHint`) чинятся порталом в **R9, задача 2** — без
него `zIndex` карточки проблему подсказок не решает.

### Задача 2. `Reports/components/ReportsToolbar.tsx`

Правый кластер получает предсказуемую геометрию:

```tsx
// Ширины фиксированы, чтобы кластер не ездил при смене периода и языка.
<div style={{ display: 'flex', alignItems: 'center', gap: '10px', marginLeft: 'auto', flexWrap: 'nowrap' }}>
```

- кнопка периода: `minWidth: 62px`, `justifyContent: 'center'`, `flex: '0 0 auto'`
  (в en «Month» короче «Месяца» — без `minWidth` селектор дышит при смене языка);
- `ComparisonBadge`: `minWidth: 172px`, `textAlign: 'center'`, `flexShrink: 0`,
  `fontVariantNumeric: 'tabular-nums'` — цифры одной ширины, «23.07» и «11.11»
  занимают одинаково;
- слот дат `custom` резервируется всегда: контейнер `width: 236px; flexShrink: 0`,
  содержимое рендерится только при `period === 'custom'` (пустой слот вместо
  прыжка на 236px);
- бейдж сравнения не исчезает и не появляется — при отсутствии
  `comparisonRange` рендерится тот же контейнер с `visibility: 'hidden'`.

Тулбар остаётся `justify-content: space-between`: вкладки слева, кластер
справа — но теперь его ширина константна и вкладки не двигаются.

### Задача 4. Вкладка в URL + «Активные клиенты»

**`Reports.tsx`:** `activeTab` переезжает в searchParams — ссылка на вкладку
шарится, кросс-табная навигация начинает работать.

```tsx
const [searchParams, setSearchParams] = useSearchParams();
const activeTab = (TABS.includes(searchParams.get('tab') as Tab) ? searchParams.get('tab') : 'overview') as Tab;
const setActiveTab = (tab: Tab) => setSearchParams(prev => {
  const next = new URLSearchParams(prev);
  next.set('tab', tab);
  next.delete('segment');            // хвост от прошлой вкладки не тащим
  return next;
}, { replace: true });
```

`useReportFilters` уже живёт в тех же searchParams — период и фильтры при смене
вкладки сохраняются, это же поведение и нужно.

**`OverviewTab.tsx:165`** (KPI «Активные клиенты»):

```tsx
onClick={() => navigate(`/dashboard/reports?${new URLSearchParams({
  ...Object.fromEntries(searchParams), tab: 'clients', segment: 'active',
})}`)}
```

**`ClientsTab.tsx`:** читает `segment` из URL и открывает drilldown с
**правильным заголовком** — `t('clients.segments.active.name')` = «Активные
клиенты» (сейчас на вкладке нет ни такого раздела, ни такого названия — это и
есть суть пункта 22):

```tsx
useEffect(() => {
  const seg = searchParams.get('segment');
  if (seg) setDrilldown({ kind: 'segment', key: seg, title: t(`clients.segments.${seg}.name`) });
}, [searchParams, t]);
```

Запрос уходит в существующий `analyticsApi.getClientsReportSegment('active', params)`
(ключ поддержан бэком в R6). Колонки drilldown те же, `value` показывает число
визитов за период — подпись колонки для этого сегмента: «Визитов».

`ClientDynamicsCard` (`OverviewTab.tsx:236`) тоже ведёт на `tab=clients` — после
этой задачи навигация наконец сработает; сегмент не задаём, открывается вкладка
целиком.

## Styling & UI/UX

- Карточки: подъём остаётся 2px + мягкая тень — премиальность не трогаем,
  чиним только слои и плавность.
- Тулбар: цифры дат — `tabular-nums`; никакого «схлопывания» бейджа сравнения.
- Переключатель периода: активная кнопка белая с тенью `0 1px 6px` —
  как сейчас, но с фиксированной шириной.
- Переход между вкладками сохраняет анимацию `fadeSlide 0.22s` (`Reports.tsx:61`),
  ключ анимации — `activeTab` из URL.

## Edge Cases

| Ситуация | Поведение |
|---|---|
| `?tab=` с мусором (`?tab=hello`) | Фолбэк на `overview`, без падения |
| Прямая ссылка `?tab=clients&segment=active` | Вкладка Клиенты + сразу открытая модалка «Активные клиенты» |
| Сегмент `active` пуст | Модалка не открывается — Toast из R8 («В сегменте нет клиентов»), параметр `segment` из URL вычищается |
| Кнопка «Назад» браузера | `setSearchParams(..., { replace: true })` не плодит историю на каждый клик по вкладке; возврат уводит на страницу до Отчётов |
| Наведение мышью между двумя KPI подряд | Обе карточки анимируются независимо, `zIndex` активной = 3, наложения нет |
| Клик по карточке прибыли дважды | График остаётся на `profit`, повторный клик — no-op |
| Период `custom` без выбранных дат | Слот дат уже зарезервирован, ширина не меняется; запрос не уходит, пока обе даты пусты |
| Язык переключён на en в открытой вкладке | Ширина кластера не меняется (`minWidth`), тулбар не прыгает |

## Definition of Done

- Переключение день → неделя → месяц → год: вкладки и кнопки тулбара стоят
  на месте, пиксель в пиксель (проверка скриншотами до/после).
- Наведение по ряду KPI слева направо: ни одна карточка не перекрывается
  соседней, подсказки видны целиком, движения плавные.
- Клик по «Прибыль» переключает график и никуда не уводит.
- Клик по «Активные клиенты» открывает раздел с заголовком «Активные клиенты»,
  число строк совпадает со значением KPI.
- `npm run build && npm run lint` зелёные.
