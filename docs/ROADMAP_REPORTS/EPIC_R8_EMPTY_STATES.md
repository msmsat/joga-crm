# EPIC R8 — Empty States: красивое «данных пока нет»

**Цель:** новая студия и пустой период перестают выглядеть как сломанный
продукт. Один компонент кита `EmptyState` — на всю страницу, на виджет и на
таблицу; клик по пустому списку сегментов честно говорит, что списка нет, а не
открывает пустую модалку.

## Backend Architecture

Изменений нет. Признак пустоты вычисляется на фронте по уже приходящим данным —
отдельного флага `is_empty` в DTO **не заводим**: он дублировал бы то, что и
так видно в ответе, и его пришлось бы поддерживать в пяти эндпоинтах.

## Frontend Architecture

### Задача 1. `front/src/components/ui/EmptyState.tsx` (новый компонент кита)

Правило CLAUDE.md §5: новый общий компонент → в `components/ui/`, экспорт из
`index.ts`, **строка в таблицу UI-кита в CLAUDE.md**.

```tsx
export interface EmptyStateProps {
  title: string;
  text?: string;
  icon?: 'chart' | 'clients' | 'calendar' | 'money' | 'search';  // inline SVG, эмодзи запрещены
  action?: React.ReactNode;      // обычно <Button variant="primary" size="sm">
  size?: 'sm' | 'lg';            // sm — внутри виджета, lg — на всю вкладку
}
```

- Иллюстрация — inline SVG 96×96 (`lg`) / 48×48 (`sm`): тонкие линии
  `stroke="currentColor"`, цвет `rgba(249,160,139,0.55)`, внутри —
  мягкий персиковый круг-подложка `rgba(249,160,139,0.08)`.
- Анимация входа — `framer-motion` (в проекте установлен, §5 CLAUDE.md):
  `initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.32, ease: [0.34, 1.1, 0.64, 1] }}`; иллюстрация —
  отдельный `motion.div` с задержкой `0.06` и `scale: 0.94 → 1`.
- Типографика: `title` 15px/800 `var(--text)`, `text` 13px/500 `var(--text3)`,
  `max-width: 320px`, центр, `padding` 48px (`lg`) / 20px (`sm`).
- Никаких рамок и линеек — только воздух (ДС: негативное пространство вместо
  разделителей).

Экспорт в `front/src/components/ui/index.ts`; `EmptyTab.tsx` после этого —
обёртка над `EmptyState`, а не своя вёрстка (или удаляется совсем).

### Задача 2. Пустое состояние вкладки (аудит 10)

**Новый хелпер:** `Reports/hooks/useIsEmpty.ts`

```ts
/** Данные пришли, но смотреть не на что: все KPI по нулям и все списки пусты. */
export function isAllZero(values: number[], lists: unknown[][]): boolean {
  return values.every(v => !v) && lists.every(l => !l.length);
}
```

Подключение в каждой вкладке — до основной разметки:

| Вкладка | Условие пустоты |
|---|---|
| `OverviewTab` | KPI revenue/profit/attendance/active_clients/fill_rate = 0 **и** `revenue_structure` пуст |
| `SalesTab` | KPI revenue/sales_count = 0 **и** `products` пуст |
| `ClientsTab` | KPI new/returned/lost = 0 **и** все `risk_segments`/`loyal_segments` по нулям |
| `TeamTab` | `trainers` пуст |
| `ScheduleTab` | `heatmap` пуст (ни одного занятия в периоде) |

Рендер: `<EmptyState size="lg" icon=… title={t('empty.noData.title')}
text={t('empty.noData.text')} action={<Button size="sm" variant="ghost"
onClick={() => setPeriod('year')}>{t('empty.noData.widen')}</Button>} />`
— кнопка расширяет период до года, это единственное осмысленное действие
владельца в такой момент.

**Важно:** пустое состояние показывается только когда запрос **завершён**
(`!isFetching || data`). Во время первой загрузки — ничего (у Query уже стоит
`placeholderData: prev => prev`), иначе экран моргнёт «нет данных → есть данные».

Тулбар (период, фильтры, экспорт) **остаётся на экране** — иначе владелец не
сможет сменить период, чтобы данные появились.

### Задача 4. Toast на пустом сегменте (аудит 12)

`ClientsTab.tsx`, функция `openSegment(key)` (строка 93). Сейчас модалка
открывается всегда и показывает пустую таблицу.

```ts
const { showToast } = useToast();                  // из components/ui/index
const segmentCount = (key: string) =>
  [...(data?.risk_segments ?? []), ...(data?.loyal_segments ?? [])]
    .find(s => s.key === key)?.count ?? 0;

const openSegment = (key: string) => {
  if (!segmentCount(key)) {
    showToast(t('clients.emptySegment', { name: t(`clients.segments.${key}.name`) }), 'info');
    return;
  }
  setDrilldown({ kind: 'segment', key, title: t(`clients.segments.${key}.name`) });
};
```

Считаем по уже загруженному `data` — лишнего запроса нет. Тот же приём для
`SlicesCards` и строк тепловой карты: клик по слоту без занятий уже
заблокирован (`disabled`), менять не нужно.

## Styling & UI/UX

- Иллюстрация никогда не кричит: контур, а не заливка; персиковый на 8–55 %
  прозрачности, серый текст.
- `size="lg"` — по центру карточки высотой не меньше 320px, чтобы пустая
  вкладка не выглядела «обрезанной».
- Тёмная тема: цвета только из токенов (`var(--text3)`, `var(--border)`),
  иллюстрация через `currentColor` — hex в разметке запрещён.
- Toast — существующий `useToast` кита, вариант `info`, 3 секунды.

## Edge Cases

| Ситуация | Поведение |
|---|---|
| Первая загрузка вкладки | Empty State не показывается — ждём ответ, показываем предыдущие данные (Query `placeholderData`) |
| Данные есть, но один виджет пуст | Только этот виджет получает `size="sm"`; вкладка живёт как обычно |
| Все KPI = 0, но список тренеров не пуст | Это **не** пустое состояние: тренеры есть, просто без выручки — рисуем таблицу |
| Ошибка запроса (500/сеть) | Empty State не подходит — это `ErrorBoundary`/сообщение об ошибке; молча подменять ошибку на «нет данных» запрещено |
| Пустой сегмент + двойной клик | Toast не дублируется (у `useToast` уже дедуп по тексту; если нет — гасим повтор в течение 2 с) |
| Экспорт CSV при пустых данных | Кнопка активна, файл содержит только заголовки — падать нельзя |
