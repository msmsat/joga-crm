# Velora CRM — Техническое задание (функционал по страницам)

Документ отвечает на вопрос «что должен уметь продукт». Общие сведения о продукте, целях и ролях — в `CLAUDE.md`.
Пометка **💡 предложение:** — идея на улучшение, требует утверждения владельцем продукта.

---

## 1. Каркас приложения

Доступ: все роли (внутри — ограничения по ролям).

### 1.1 Боковое меню

Пользователь может перейти в любой доступный ему раздел.
Пользователь может открыть Журнал закреплённой кнопкой внизу.
Пользователь может открыть свой Профиль по пилюле профиля внизу.

Меню содержит:
- логотип Velora и название текущей студии;
- пункты: Дашборд, Сотрудники, Каталог, Клиенты (бейдж с количеством), Отчёты, Онлайн-запись, Финансы, Уведомления (бейдж с числом новых), Лояльность, Velora AI;
- разделитель;
- пункты: Настройки, Тариф и оплата;
- закреплённую кнопку «Журнал» с подписью «Расписание занятий»;
- пилюлю профиля: аватар с инициалами, email.

При роли администратор или тренер:
- в меню видны только доступные разделы, остальные пункты скрыты.

### 1.2 Верхняя панель

Пользователь может видеть заголовок и подзаголовок текущего раздела.
Пользователь может открыть глобальную AI-панель кнопкой «AI».
Пользователь может нажать «+ Создать» — откроется создание записи в Журнале.

Панель содержит:
- заголовок и подзаголовок раздела (слева);
- AI-строку (по центру);
- кнопку «AI» и тёмную кнопку «+ Создать» (справа).

#### 1.2.1 AI-строка

Пользователь может ввести вопрос ассистенту, не покидая страницу.
Пользователь может отправить вопрос клавишей Enter.
Пользователь может закрыть панель ответа клавишей Esc или кликом мимо.

AI-строка содержит:
- поле с плейсхолдером «Спросите AI или поставьте задачу...»;
- иконку-искру;
- подсказку «Enter» (при открытой панели — «Esc»).

После отправки вопроса:
- под строкой плавно выпадает панель с эхо-строкой вопроса;
- показывается состояние «Velora AI думает над ответом» (пульсирующая иконка, бегущие точки);
- затем ответ печатается по буквам с мигающим курсором.

При фокусе:
- строка подсвечивается акцентным свечением и слегка приподнимается.

### 1.3 Глобальная AI-панель

Пользователь может открыть панель с любой страницы кнопкой «AI».
Пользователь может закрыть панель крестиком.
Пользователь может начать новый диалог кнопкой «+ Новый чат».
Пользователь может открыть историю диалогов и продолжить любой из них.
Пользователь может отправить вопрос из поля ввода (Enter — отправить, Shift+Enter — новая строка).
Пользователь может отправить готовый вопрос кликом по подсказке-чипу.

Панель содержит:
- шапку: кнопка «История чатов», заголовок «Velora AI», кнопка «+ Новый чат», крестик;
- пустое состояние: заголовок «Чем могу помочь?», подпись «Задайте любой вопрос о вашей студии, клиентах или бизнесе», 4 чипа-подсказки: «Анализ выручки», «Шаблон письма», «Идеи для акции», «Отчёт по клиентам»;
- ленту сообщений (пузыри пользователя и ассистента);
- поле ввода «Спросите что-нибудь…» с кнопкой отправки.

При ответе ассистента:
- показывается индикатор «Velora AI думает» с тремя точками;
- ответ печатается по буквам;
- лента автоматически прокручивается вниз;
- поле ввода блокируется, пока ассистент думает.

Экран истории содержит:
- кнопку «‹ Чаты» (назад), заголовок «История»;
- карточки диалогов: заголовок, превью, относительное время («N мин/ч/дн назад»), число сообщений.

### 1.4 Общее

Пользователь может переключать светлую и тёмную тему.
Пользователь может работать с интерфейсом на разных языках.

Ограничение доступа:
- каждая роль имеет доступ только к своим разделам и данным;
- ограничение действует и в интерфейсе, и на сервере — получить чужие данные в обход интерфейса нельзя.

## 2. Авторизация и вход

Доступ: публичная часть (единственная без входа). Гостевого режима нет — продукт доступен только после регистрации.

### 2.1 Регистрация

Пользователь может зарегистрироваться по email и паролю.
Пользователь может зарегистрироваться через Google.

Экран регистрации содержит:
- поле Email;
- поле Пароль;
- кнопку «Зарегистрироваться»;
- кнопку «Продолжить через Google».

После успешной регистрации:
- пользователю отправляется письмо с кодом подтверждения;
- пользователь вводит код и подтверждает email;
- после подтверждения открывается онбординг.

### 2.2 Вход

Пользователь может войти по email и паролю.
Пользователь может войти через Google.
Пользователь может восстановить забытый пароль.
Пользователь может изменить пароль на отдельной странице.

Экран входа содержит:
- поле Email;
- поле Пароль;
- кнопку входа;
- кнопку «Продолжить через Google»;
- ссылку «Забыли пароль?».

После запроса восстановления:
- пользователю отправляется письмо со ссылкой на сброс пароля;
- по ссылке открывается страница задания нового пароля.

### 2.3 Онбординг

Пользователь проходит онбординг из 5 шагов сразу после регистрации.
Без онбординга работа в CRM не начинается.

Шаги онбординга:
1. название и тип бизнеса;
2. направления занятий;
3. контакты и адрес;
4. часы работы;
5. региональные настройки: часовой пояс, валюта, язык.

После завершения онбординга:
- создаётся студия, пользователь становится её владельцем;
- открывается Дашборд.

### 2.4 Несколько студий

Пользователь может состоять в нескольких студиях.
Пользователь может выбрать студию при входе.
Пользователь может переключаться между студиями (см. 14.10).

## 3. Дашборд (Обзор)

Доступ: все роли.

### 3.1 Ряд метрик

Пользователь может видеть 4 ключевые метрики студии.
Пользователь может кликнуть по метрике — она становится активной и переключает большой график на свои данные.
Пользователь может нажать «Подробнее ↗» у активной метрики — переход в соответствующий раздел.

Ряд содержит 4 карточки:
- «Выручка за месяц» (₽) — переход в Финансы;
- «Активных клиентов» — переход в Клиенты;
- «Записей сегодня» — переход в Онлайн-запись;
- «Уровень удержания» (%) — переход в Отчёты.

Каждая карточка содержит:
- иконку, заголовок, значение;
- строку тренда к прошлому периоду (например «↑ 18.4% vs прошлый мес.», «↑ 12 новых»);
- подсветку цветом метрики у активной.

### 3.2 График аналитики

Пользователь может переключать период графика: «Нед.» / «Мес.» / «Год».
Пользователь может навести курсор на столбец — столбец подсвечивается и показывается тултип со значением.

График содержит:
- заголовок из названия активной метрики и периода (например «Выручка по неделям»);
- подзаголовок («Последние 8 недель/месяцев/лет»);
- 8 столбцов, последний (текущий период) выделен цветом метрики;
- подписи под столбцами: даты недель / месяцы / годы.

Тултип показывает значение в формате метрики: «₽…K» / «… чел.» / «… зап.» / «…%».

### 3.3 Виджет «Задачи на сегодня»

Пользователь может отметить задачу выполненной (и снять отметку) кликом по строке.
Пользователь может раскрыть блок «Выполнено · N» со свёрнутыми выполненными задачами.
Пользователь может добавить задачу кнопкой «+ Добавить задачу».

Виджет содержит:
- заголовок «Задачи на сегодня», счётчик «N задач осталось», бейдж с числом невыполненных;
- строки задач: чекбокс, текст, цветной тег категории (Клиент / Финансы / Лиды / Отчёты / Журнал / Персонал), точку приоритета с тултипом «Высокий/Средний/Низкий приоритет»;
- блок «Выполнено · N» (задачи зачёркнуты и приглушены);
- кнопку «+ Добавить задачу».

Форма добавления содержит:
- поле «Название задачи...» (Enter — добавить);
- селект категории (по умолчанию «Клиент»);
- селект приоритета: «Низкий» / «Средний» / «Высокий» (по умолчанию «Средний»);
- кнопки «Отмена» и «Добавить» (неактивна при пустом поле).

После добавления:
- задача появляется сверху списка с анимацией.

### 3.4 Карточка «Последние события»

Пользователь может видеть ленту последних событий студии.
Пользователь может перейти в Журнал по ссылке «Смотреть все →».
Пользователь может открыть меню «⋯» у события и выполнить действие.

Каждое событие содержит:
- цветной маркер типа;
- текст (имя клиента / сумма выделены жирным);
- относительное время: «только что» / «N мин. назад» / «N ч назад» / «N дн. назад»;
- кнопку «⋯».

Меню «⋯» содержит (набор зависит от типа события):
- «Открыть профиль» — всегда;
- «Написать в WhatsApp» — для записи;
- «Отправить чек» — для оплаты;
- «Отменить запись» (красным) — для записи или отмены.

### 3.5 Карточки-сводки

Пользователь может видеть три сводки внизу страницы.

Сводки содержат:
- «Топ услуги» — 4 услуги с процентом популярности и полосой прогресса;
- «Нагрузка тренеров» — тренеры с процентом загрузки и полосой;
- «Кассы (сегодня)» — суммы по основной кассе, безналу и онлайн-платежам, строка «Итого за день».

## 4. Журнал

Доступ: владелец и администратор — всё расписание; тренер — только свои занятия.

### 4.1 Тулбар (верхняя панель 1)

Пользователь может листать дни кнопками «‹» и «›» (в режиме «Неделя» — по 7 дней).
Пользователь может ввести дату вручную, кликнув по кнопке-дате.
Пользователь может вернуться к текущей дате кнопкой «Сегодня».
Пользователь может переключить вид колонок: «Тренеры» / «Залы».
Пользователь может включать и выключать колонки фильтрами-пилюлями.
Пользователь может переключить период: «День» / «Неделя». 💡 предложение: позже добавить «Месяц» — переключатель уже спроектирован на 3 позиции.

Тулбар содержит:
- кнопки «‹» и «›»;
- кнопку-дату с иконкой календаря (например «18 Июнь 2026»);
- кнопку «Сегодня»;
- переключатель «Тренеры» / «Залы»;
- фильтры-пилюли: инициалы тренеров в их цветах (в виде «Тренеры») или названия залов (в виде «Залы»);
- переключатель «День» / «Неделя».

При ручном вводе даты:
- принимаются форматы через точку, слэш, тире и пробел; короткий год («26» → 2026);
- Enter — применить, Esc — отмена;
- при неверной дате показывается уведомление «Неверный формат даты».

При работе с фильтрами:
- клик включает/выключает колонку;
- последнюю активную колонку выключить нельзя.

### 4.2 Сводка дня и режим черновика (верхняя панель 2)

Пользователь может видеть 5 цифр текущего дня.
Пользователь может включить режим черновика кнопкой «Изменить график».
Пользователь может отменить изменения кнопкой «Отменить».
Пользователь может опубликовать изменения кнопкой «Опубликовать».
Пользователь может выбрать шаг сетки: 1 / 5 / 15 минут (в режиме черновика).

Панель содержит:
- показатели: «N занятий сегодня», «N записей клиентов», «N% средняя загрузка», «N ожидает подтверждения» (подсвечивается, если больше 0), «N тренеров активно»;
- индикатор режима: «Рабочий график» или «Режим черновика»;
- в обычном режиме — кнопку «Изменить график»;
- в черновике — кнопки «Отменить», «Опубликовать» и переключатель «Шаг сетки».

После нажатия «Опубликовать»:
- режим черновика выключается;
- показывается уведомление «График успешно опубликован!»;
- изменения становятся видимыми и уходят в уведомления.

Смысл режима: все изменения расписания делаются в черновике и публикуются одним действием.

### 4.3 Сетка расписания

Пользователь может видеть расписание дня: колонки — тренеры или залы, строки — часы (07:00–21:00).
Пользователь может видеть недельный вид: колонки — 7 дней (Пн–Вс), сегодняшний день выделен.

Заголовок колонки содержит:
- в виде «Тренеры»: аватар с инициалами в цвете тренера, полное имя, специализацию, строку «N занятий · M чел.»;
- в виде «Залы»: название зала и «N занятий на сегодня»;
- в виде «Неделя»: день недели, число (сегодня — залито акцентом), «N зан. · M чел.», «Загрузка: K%».

Карточка занятия содержит:
- название занятия;
- при достаточной высоте — «записано/мест»;
- бейдж «FULL» при полной группе;
- тонкую полосу заполненности внизу;
- цвет тренера;
- статус: подтверждено / ожидает / отменено.

При прокрутке:
- заголовки колонок сжимаются в компактный вид.

При пересечении занятий:
- карточки автоматически раскладываются рядом, не накладываясь.

### 4.4 Жесты в сетке (работают в режиме черновика)

Пользователь может создать занятие, нажав на свободную ячейку.
Пользователь может перенести занятие перетаскиванием в другую ячейку.
Пользователь может изменить длительность занятия, потянув его за верхний или нижний край.

При создании:
- наведение подсвечивает свободную ячейку;
- нажатие открывает окно «Новое занятие», а в сетке появляется живое пульсирующее превью будущей карточки.

При переносе:
- в виде «День» меняется время и тренер/зал;
- в виде «Неделя» меняется дата занятия;
- время привязывается к выбранному шагу сетки;
- после переноса показывается уведомление «Занятие перенесено»;
- при попытке перетащить вне черновика — подсказка «Для перемещения занятий включите режим черновика».

При растягивании за края:
- у выбранного занятия появляются зоны захвата сверху и снизу;
- верхний край меняет время начала, нижний — время окончания;
- во время растягивания показываются подсказки с новым временем и «призрак» исходных границ;
- минимальная длительность — один шаг сетки, в пределах рабочего дня;
- после — уведомление «Время начала изменено» / «Время окончания изменено».

### 4.5 Попап занятия (по клику на карточку)

Пользователь может открыть детали занятия кликом по карточке.
Пользователь может добавить клиента на занятие кнопкой «Добавить» (доступно всегда).
Пользователь может изменить занятие кнопкой «Изменить» (в черновике).
Пользователь может удалить занятие красной кнопкой «Удалить занятие» (в черновике). 💡 предложение: удаление — с подтверждением, если на занятие уже записаны клиенты.

Попап содержит:
- шапку: название занятия, время «начало – конец», бейдж статуса «Подтверждено» / «Ожидает»;
- «записано/мест», зал, полное имя тренера;
- блок «Заполненность» с процентом и полосой;
- кнопки «Добавить», «Изменить», «Удалить занятие».

Режим редактирования (внутри попапа) содержит:
- поле «Название занятия»;
- поля времени «С» и «До» с выпадающими списками (шаг = шаг сетки);
- чипсы выбора зала;
- поле максимума мест;
- кнопки «Отмена» и «Сохранить».

После сохранения — уведомление «Занятие обновлено».
После удаления — уведомление «Занятие удалено».

### 4.6 Окно «Новое занятие»

Пользователь может создать занятие, заполнив форму.

Окно содержит:
- заголовок «Новое занятие» и подпись «Время слота: НАЧАЛО – КОНЕЦ»;
- поле «Название программы» (плейсхолдер «Например: Пилатес Реформер»);
- выбор «Локация / Зал» чипсами;
- поля «Начало» и «Конец» (списки времени с шагом сетки);
- поле «Лимит группы» (1–50, по умолчанию 8);
- выбор «Назначить тренера» — список тренеров с аватарами и галочкой у выбранного;
- кнопки «Отмена» и «Создать занятие».

При заполнении:
- конец не может быть раньше начала (автокоррекция);
- создание невозможно при пустом названии.

После создания — уведомление «Занятие добавлено».

### 4.7 Окно «Добавить клиента» (на занятие)

Пользователь может найти клиента по имени или телефону.
Пользователь может выбрать нескольких клиентов сразу.
Пользователь может добавить выбранных кнопкой «Добавить (N)».

Окно содержит:
- заголовок «Добавить клиента» и подпись «Название занятия · время»;
- поиск «Поиск по имени или телефону...»;
- список клиентов: аватар, имя и фамилия, телефон, «N визитов», галочка у выбранных;
- пустое состояние «Клиенты не найдены»;
- футер: счётчик «Выбрано: N» (или «Выберите клиентов из списка»), кнопки «Отмена» и «Добавить (N)» (неактивна без выбора).

После добавления:
- уведомление «N клиент(а) добавлено»;
- число записанных не может превысить лимит мест.

### 4.8 Правая панель

## 5. Клиенты

Доступ: владелец и администратор — все клиенты; тренер — только свои.

### 5.1 Тулбар

Пользователь может фильтровать клиентов по категориям.
Пользователь может искать клиента по имени, фамилии, телефону, email и тегам.
Пользователь может открыть мастер добавления кнопкой «Добавить клиента».

Тулбар содержит:
- вкладки-категории со счётчиками: «Все», «VIP», «Активные», «Новые», «С абонементом», «Неактивные», «День рождения»;
- поиск «Поиск клиента...»;
- кнопку «Добавить клиента».

### 5.2 Сетка карточек клиентов

Пользователь может открыть профиль клиента кликом по карточке (повторный клик — закрыть).

Клиенты показываются карточками-плитками. Карточка содержит:
- аватар с инициалами в цвете клиента;
- имя и фамилию;
- строку «N визитов · дата последнего визита»;
- бейдж статуса: «Активный» / «VIP» / «Новый», у замороженных — «❄ заморожен»;
- «Потрачено всего» — сумму;
- строку «Абонемент» — «X из Y» занятий с полосой прогресса (при малом остатке — тревожный цвет, у замороженных — голубой);
- уровень лояльности (Bronze / Silver / Gold / Platinum — по баллам) и количество баллов.

При удалении клиента:
- карточка анимированно исчезает.

### 5.3 Мастер «Добавить клиента» (4 шага)

Пользователь может добавить клиента, пройдя 4 шага.
Пользователь может вернуться на шаг назад кнопкой «← Назад».

Мастер содержит:
- слева — брендовую панель: номер «Шаг N из 4», заголовок и подзаголовок шага, индикатор прогресса, подпись «Данные защищены и видны только администраторам»;
- справа — форму с заголовком «Новый клиент» и крестиком;
- футер: «Отмена» / «← Назад», индикатор «N / 4», кнопку «Продолжить →» или «Добавить клиента ✓».

#### 5.3.1 Шаг 1 «Личные данные»

Шаг содержит:
- живой предпросмотр аватара «Новый клиент · Предпросмотр»;
- поле «ФИО *» (плейсхолдер «Иван Иванов»);
- поле «Телефон» («+7 900 000-00-00»);
- поле «Email» («client@example.com»).

При заполнении:
- имя обязательно, минимум 2 символа — иначе ошибка «Введите имя (минимум 2 символа)» и кнопка «Продолжить» неактивна;
- телефон, если введён, — минимум 6 цифр («Некорректный номер телефона»);
- email должен содержать «@» («Некорректный email»).

#### 5.3.2 Шаг 2 «Детали профиля» (необязательно)

Шаг содержит:
- поле «День рождения» («12 апреля 1990»);
- поле «Город» («Москва»);
- поле «Теги» (плейсхолдер «Пилатес, VIP, Реабилитация… (Enter)», добавление по Enter).

#### 5.3.3 Шаг 3 «Абонемент и заметка»

Пользователь может выбрать количество занятий абонемента.
Пользователь может написать заметку.

Шаг содержит:
- «Количество занятий» — три плитки: 8 / 10 / 12 (у выбранной подпись «Выбрано»);
- «Заметка» — текстовое поле «Особые пожелания, противопоказания, предпочтения...».

#### 5.3.4 Шаг 4 «Готово»

Шаг содержит:
- отметку «Данные заполнены!»;
- сводную карточку: Телефон, Email, Д.р., Город, Абонемент (N занятий), Теги;
- переключатель «Отправить SMS-приветствие» (по умолчанию включён) с подписью «Клиент получит сообщение с приглашением»;
- зелёную кнопку «Добавить клиента ✓».

После добавления:
- список клиентов и счётчики категорий обновляются.

### 5.4 Панель профиля клиента (выезжает справа)

#### 5.4.1 Шапка

Пользователь может сменить статус клиента из выпадающего списка: «Активный» / «VIP» / «Новый» / «Неактивный» / «Заморожен».
Пользователь может изменить дату регистрации, кликнув по ней (Enter — сохранить, Esc — отмена).
Пользователь может закрыть панель крестиком.

Шапка содержит:
- аватар, имя и фамилию;
- бейдж «❄ Заморожен» (если применимо);
- кликабельный статус-бейдж со стрелкой;
- дату регистрации «с …»;
- крестик закрытия.

#### 5.4.2 Быстрые действия

Пользователь может позвонить клиенту кнопкой «Позвонить».
Пользователь может написать клиенту кнопкой «Сообщение».
Пользователь может записать клиента на занятие кнопкой «Записать».
Пользователь может начислить бонус кнопкой «Бонус».

Ряд содержит 4 кнопки: «Позвонить» (зелёная), «Сообщение» (синяя), «Записать» (акцентная), «Бонус» (жёлтая). Панели действий взаимоисключающие.

Панель «Сообщение клиенту» содержит:
- текстовое поле «Введите сообщение...»;
- кнопку «Отправить сообщение».
После отправки — уведомление «Сообщение отправлено».

Панель «Быстрая запись» содержит:
- карусель дат на 7 дней вперёд (первая — «Сег»);
- сетку слотов времени (09:00–20:00);
- выпадающий выбор услуги «Выберите услугу...»;
- кнопку «Записать →».
После записи — уведомление «Запись создана».

Панель «Начислить бонус» содержит три варианта:
- «+500 баллов» — «Начислить на баланс лояльности»;
- «Скидка 10% на продление» — «На следующую покупку абонемента»;
- «Подарочное занятие» — «Бесплатное дополнительное занятие».
После выбора — уведомление «Бонус начислен: …».

#### 5.4.3 Вкладка «Профиль»

Пользователь может скопировать телефон и email кликом (подпись «нажмите для копирования»).
Пользователь может раскрыть детальную активность кликом по карточке активности.
Пользователь может переключить период активности: 1 / 3 / 6 мес.
Пользователь может добавить тег (из подсказанных или свой по Enter) и удалить тег крестиком.
Пользователь может заморозить/разморозить абонемент кнопкой «Заморозить» / «Разморозить».
Пользователь может удалить клиента красной кнопкой «Удалить».

Вкладка содержит:
- секцию «Контакты»: Телефон, Email, День рождения, Город;
- 3 плитки: «Визитов», «Потрачено», «Абонемент» (X/Y);
- блок лояльности: кольцевой индикатор баллов, уровень, «До следующего: N баллов», множитель «×1.0 / ×1.5 / ×2.0 к баллам»;
- карточку «Абонемент»: «X/Y занятий», полоса и точки по занятиям; при малом остатке — подсказка «Осталось мало занятий — предложить продление»;
- карточку «Активность (3 мес.)»: свёрнуто — мини-график и «Нажмите для деталей»; раскрыто — число визитов за период, переключатель периода, столбчатый график с тултипами «N визитов» и текстовый вывод (например «Устойчивый темп посещений — низкая вероятность оттока»);
- секцию «Теги»: теги с крестиками, кнопку «Добавить», подсказанные теги (Пробное, VIP, Рекомендация, Онлайн, Групповой, Персональный), поле «Свой тег + Enter...»;
- блок «Заморозить» / «Удалить».

После копирования — уведомление «Успешно скопировано».

#### 5.4.4 Вкладка «События»

Пользователь может фильтровать события: «Все» / «Оплаты» / «Посещения» / «Заморозки».

Вкладка содержит:
- фильтры-вкладки;
- список событий: иконка типа, заголовок, «тренер · дата», сумма/значение (оплаты — зелёным, заморозки — голубым);
- пустое состояние «Нет событий».

#### 5.4.5 Вкладка «Заметки»

Пользователь может добавить заметку кнопкой «Добавить заметку».
Пользователь может отредактировать заметку по кнопке-карандашу (Ctrl+Enter — сохранить).
Пользователь может удалить заметку.

Вкладка содержит:
- заголовок «Заметки администратора»;
- список заметок: подпись «Заметка · дата», текст, кнопку-карандаш;
- форму добавления: текстовое поле «Введите заметку...», кнопки «Сохранить» / «Отмена».

После добавления — заметка помечается «только что».

### 5.5 Удаление клиента

Пользователь может удалить клиента только через окно подтверждения. 💡 предложение: сейчас кнопка в панели профиля удаляет мимо подтверждения — исправить.

Шапка профиля содержит:
- аватар, имя и фамилию, должность, статус онлайн/оффлайн;
- кнопки «Написать», «Позвонить», «Редактировать»;
- чипы: «Активен», email, телефон;
- полосу «Загрузка» с процентом (высокая — зелёная, средняя — акцентная, низкая — розовая).

- Остеопат: Выручка · Рейтинг · Приёмов · Завершено.

- Администратор: Записей · Обработано · Касса · Загрузка.
- Управляющий: Выручка · Записей · Посещений · Загрузка.
- Владелец: Выручка · Посещений · Загрузка · Сотрудников.

Форматы значений: деньги — «₽NK», проценты — «N%», рейтинг — «N★»; при отсутствии данных — «—».

- легенду залов по цветам + «Свободно».

Вид «Месяц» содержит:
- тепловую карту дней месяца (интенсивность = число занятий);
- легенду «Свободно» / «Занято».

Блок «Расписание на сегодня» содержит:
- кнопку «Добавить»;
- занятия: время начала, длительность в минутах, цветная точка зала, название, «N клиентов · зал», бейдж зала, крестик отмены.

При отмене занятия:
- показывается окно подтверждения с чекбоксом «Больше не спрашивать»;

Владелец может добавить сотрудника, пройдя 4 шага.

💡 предложение: здесь же селект «Роль доступа»: Администратор / Тренер — два варианта с одной строкой пояснения (см. «Целевая аудитория» в CLAUDE.md).

Шаг содержит:

### 6.5 Редактирование сотрудника

Владелец может изменить фото, данные, должность, услуги, зарплату и расписание.
Владелец может переключить онлайн-статус сотрудника тумблером.
Владелец может удалить сотрудника (с инлайн-подтверждением «Отмена» / «Удалить»).

Окно содержит:
- слева — живую «визитку» (аватар, имя, должность, услуги, мини-неделя, зарплата — обновляется по мере ввода), тумблер онлайн-статуса, блок удаления;
- справа — 4 вкладки.

Вкладки:
- «Профиль»: фото (загрузка/замена/удаление), «ФИО *», «Телефон», «Email»;
- «Роль»: плитки должностей + своя, услуги (пресеты + свои);
- «Зарплата»: крупный дисплей суммы, тип Фикс / Процент / Почасовая, поле суммы;
- «Расписание»: 7 дней с тумблерами и временем.

Кнопка сохранения показывает состояния: «Сохранить изменения» → «Сохранение…» → «Сохранено».

Ограничение: последнего владельца удалить нельзя — вместо кнопки показывается поясняющая плашка.

💡 предложение: достроить формы редактирования филиала и добавления/редактирования зала — кнопки в интерфейсе есть, формы не реализованы. Форма зала: название, вместимость, площадь, цвет (используется в Журнале), оборудование, стоимость часа, признак «онлайн-зал».

Шаг 3 «Фото студии» содержит:

- кнопку «Создать студию» и ссылку «Пропустить фото и создать».

Валидация: название, город и адрес — от 2 символов.

#### 8.1.1 Окно Telegram

Владелец может подключить бота, вставив токен.
Владелец может отключить бота.

- кнопку «Подключить бота» (неактивна при пустом поле);

Окно (подключён) содержит:
- галочку и бейдж «Подключён», имя бота «@…», замаскированный токен;

Владелец может скопировать ссылку записи (→ «Ссылка скопирована»).

Окно содержит:

#### 8.1.3 Окно Веб-сайт

Владелец может скопировать код-вставку кнопкой «Копировать».
💡 предложение: для MVP ограничиться простой ссылкой на страницу записи (как в Instagram), полноценный встраиваемый виджет отложить — обсудить.

Окно содержит:
- готовый код для сайта (Tilda, WordPress или любой другой);
- инструкцию, куда вставить код.

#### 8.1.4 Окно WhatsApp

Владелец может подключить WhatsApp Business сканированием QR-кода.
Владелец может отключить канал.

Окно содержит три состояния:
- стартовое: описание автоответчика, анимированный макет телефона с примером переписки (клиент: «Хочу записаться» → бот отвечает ссылкой), кнопка «Подключить WhatsApp Business»;
- подключение: QR-код, текст «Отсканируйте QR-код», инструкция «WhatsApp Business → Настройки → Связанные устройства», индикатор «Ожидание подключения…»;
- подключён: галочка, бейдж «Подключён», номер, строки «Автоответ активен», «Ссылка на запись отправляется мгновенно», кнопка «Отключить».

- «Онлайн-запись активна» — «Показывать виджет клиентам» (вкл).

Владелец может настроить:

#### 8.2.3 Карточка «Брендинг виджета»

#### 8.2.4 Карточка «Уведомления»

Владелец может включать/выключать автоматические сообщения клиентам (все по умолчанию вкл):

### 8.3 Превью мини-приложения

Владелец может открыть интерактивное превью «как видит запись ваш клиент».

Превью использует текущий брендинг (цвет, логотип, язык).

## 9. Финансы

### 9.1 Счета и кассы

Владелец может видеть распределение средств по счетам.
Владелец может изменить счёт (название, баланс; для несистемных — тип).
Владелец может открыть историю счёта.
Владелец может удалить несистемный счёт.
Владелец может создать «копилку».

Вкладка содержит:
- иллюстрацию-«пончик» распределения средств: по центру «Общий капитал студии» (сумма всех балансов) и «↑ +₽N за сегодня»; легенду по счетам с долями;
- карточки счетов: цветная полоса, название, бейдж «СИСТЕМА» у системных (касса, расчётный счёт, онлайн-эквайринг), иконка типа, баланс, «+₽N сегодня»;
- плитку «Создать копилку».

Раскрытие карточки счёта содержит действия:
- «Изменить» — поля «Название», «Баланс, ₽» (только цифры), для несистемных выбор типа (Наличные / Карта / Сеть); «Сохранить» / «Отмена» (→ «Настройки сохранены»);

- «Удалить» (только несистемные) — подтверждение «Удалить эту копилку?» (→ «Счёт удалён»).

Форма «Создать копилку» содержит: «Название», «Баланс, ₽», тип; «Создать» / «Отмена» (→ «Копилка успешно создана»).

### 9.2 Операции

Владелец может искать операции («Поиск по клиентам, счетам и категориям...»).

Владелец может экспортировать операции кнопкой «Экспорт».
Владелец может раскрыть операцию, изменить её, скачать квитанцию, удалить.

Вкладка содержит:
- 3 сводки: «Приход сегодня» (+, зелёным), «Расход сегодня» (−, розовым), «Баланс дня» (±);

- строки операций: направление (↑/↓), название, бейдж «Ожидание» у незавершённых, «клиент • счёт/категория», сумма ±, дата;
- пустое состояние «Операции не найдены» + «Попробуйте изменить параметры поиска или фильтры».

Раскрытие строки содержит:
- детали: «Счёт поступления», «Категория», «Метод оплаты»;
- действия: «Изменить» (поля «Название», «Сумма, ₽», «Категория», «Дата»; → «Операция обновлена»; при пустом названии — «Введите название»), «Квитанция» (→ «Квитанция скачана»), «Удалить» (→ «Операция удалена»).

Владелец может видеть фонд зарплат и статистику.
Владелец может раскрыть тренера и изменить ставку и тип оплаты.

Вкладка содержит:
- 3 сводки: «Фонд зарплат» (текущий месяц), «Тренировок», «Средний рейтинг» (N ★);
- строки тренеров: аватар, имя и должность, метрики (Выручка ₽XK, Тренировок, Часов), бейдж ставки («₽N/ч» или «фикс.»), «₽N/мес», кнопка редактирования.

Раскрытие содержит:
- просмотр: полоса «Загрузка» %, пилюли «Рейтинг N★», «Топ-класс», «Ср. длина N мин»;
- редактирование: «Ставка (₽)», тип «Почасово» / «Фикс.», «Итого / мес (₽)» с авторасчётом «ставка × часы»; «Сохранить» / «Отмена» (→ «Зарплата обновлена»);

### 9.4 Контрагенты

- форму добавления: «Название / ФИО», «ИНН» (только цифры, до 12), «Категория», тип карточками: «Юр. лицо» («Организация или компания»), «ИП» («Индивидуальный предприниматель»), «Физ. лицо» («Частное лицо»); кнопка «Добавить контрагента» (→ «Контрагент добавлен»; при пустом названии — «Введите название»);

Раскрытие содержит:
- метрики: Сделок / Задолженность / Категория;
- действия: «Документы», «Редактировать» (инлайн, Enter/Esc; → «Изменения сохранены»), «Удалить» (подтверждение «Удалить контрагента?»; → «Контрагент удалён»).

Владелец может скачать документ, сменить его статус, удалить.

Вкладка содержит:
- 3 сводки: «Подписано», «Ждут подписи» (розовым, если > 0), «Внутренние (Без подписи)»;

Аналитика (раскрывается) содержит:

Вкладка содержит:

Владелец может создать финансовую цель.
Владелец может выбрать режим отслеживания: «Автоматически (Метрика CRM)» — прогресс двигается сам, или «Вручную (Личная копилка)» — появляется кнопка «Внести средства».
Владелец может внести средства в ручную цель.
Владелец может отредактировать и удалить цель.

Вкладка содержит:
- 3 сводки: «Цели в работе», «Выполнено», «Общий прогресс» (средний %);

При авто-режиме:

Редактирование: «Название», «Сумма, ₽», «Дедлайн»; «Сохранить» / «Отмена» (→ «Цель обновлена»).

## 10. Лояльность

Доступ: только владелец.

### 10.1 Баннер прогресса настройки

Владелец может видеть прогресс настройки программ.
Владелец может начать настройку кнопкой «Начать →» (когда не настроено ничего).

Баннер (настроено 0–4 из 5) содержит:
- кольцевой индикатор «N/5»;
- заголовок «Настройте программы лояльности»;
- подпись «Настроено N из 5 программ · Каждая программа увеличивает удержание клиентов».

Баннер (настроены все 5) содержит:
- зелёный фон, галочку, «Все программы настроены» / «Система лояльности работает в полную силу».

### 10.2 Карточки программ (5 штук)

Владелец может открыть настройку программы кликом по карточке или кнопке «⚙ Настроить».

Программы:
1. «Карты лояльности» — накопительная система баллов по уровням;
2. «Скидки и кэшбэк» — персональные предложения;
3. «Сертификаты» — подарочные и именные;
4. «Абонементы» — пакеты занятий;
5. «Реферальная» — программа «Приведи друга».

Ненастроенная карточка содержит: иконку-замок, надпись «Не настроена», кнопку «⚙ Настроить».
Настроенная карточка содержит: крупную статистику (например «89 клиентов», «34 продано»), бейдж «✓ Активна», акцентную рамку.

### 10.3 Панель настройки программы (выезжает справа)

Владелец может сохранить и активировать программу кнопкой «Сохранить и активировать».
Владелец может закрыть панель: «Отмена», крестик, Esc или повторный клик по карточке.

Панель содержит: шапку (иконка, название, «Настройка программы», крестик), настройки программы, футер («Сохранить и активировать» / «Отмена»). При открытии страница сдвигается влево.

#### 10.3.1 Карты лояльности

Настройка содержит:
- поле «Название программы» (по умолчанию «Velora Club»);
- поле «Начисление баллов (₽ за 1 балл)» (по умолчанию 100);
- уровни (просмотр): Серебро ₽0–10 000; Золото ₽10 000–50 000; Платина от ₽50 000;
- «Срок действия» баллов: 3 мес / 6 мес / 1 год (по умолчанию) / Бессрочно — с подсказкой «?» («Срок, в течение которого клиент может использовать накопленные баллы»).

#### 10.3.2 Скидки и кэшбэк

Настройка содержит:
- «Тип скидки»: «Процент (%)» (по умолчанию) / «Фиксированная (₽)» / «Кэшбэк» — меняет подпись и суффикс поля размера;
- поле размера (по умолчанию 10);
- поле «Минимальная сумма покупки» (плейсхолдер «Без ограничений»);
- «Условия применения» — чекбоксы: «Применять ко всем услугам» (вкл), «Суммировать с другими скидками» (выкл), «Показывать клиенту в личном кабинете» (выкл).

#### 10.3.3 Сертификаты

Настройка содержит:
- «Тип сертификата»: «Именной» / «Подарочный» (по умолчанию) / «На услугу»;
- для именного — поле «Имя получателя»;
- для подарочного — блок «Номиналы»: чипы ₽1 000 / ₽2 500 / ₽5 000 / ₽10 000 (мультивыбор) + «+ Добавить» (ввод своего номинала, Enter — подтвердить, Esc — отмена);
- для «на услугу» — выбор услуги чипами: Йога, Пилатес, Растяжка, Медитация, TRX;
- поле «Срок действия (дней)» (по умолчанию 365).

#### 10.3.4 Абонементы

Настройка содержит:
- «Пакеты занятий» — выбираемые строки: Мини — 8 занятий, ₽7 200, ₽900/визит; Стандарт — 12 занятий, ₽9 600, ₽800/визит; Макси — 20 занятий, ₽14 000, ₽700/визит;
- кнопку «+ Добавить пакет»;
- «Настройки» — чекбоксы: «Заморозка абонемента» (вкл), «Перенос на другого клиента» (выкл), «Автопродление» (выкл).

#### 10.3.5 Реферальная

Настройка содержит:
- поле «Бонус рефереру (₽)» (по умолчанию 500);
- поле «Скидка новому клиенту (%)» (по умолчанию 15);
- «Условие начисления» — радио: «После первого визита» (по умолчанию) / «После первой оплаты» / «Сразу при регистрации»;
- «Начисление бонуса»: «Баллами» / «На депозит» (по умолчанию) / «Скидкой».

### 10.4 Блок статистики

Владелец может навести курсор на график — показывается точка, вертикаль и тултип.

Пока не настроена ни одна программа:
- вместо графиков — плейсхолдеры «Программа не подключена» / «Данные появятся после настройки».

Блок содержит:
- карточку «Выручка через программы»: крупное значение (₽), тренд «+N% за 6 месяцев», мини-спарклайн, линейный график по месяцам (тултип «₽N»);
- карточку «Прирост клиентов»: значение «+N», тренд, спарклайн, график (тултип «N чел.»);
- «Ключевые показатели» — 4 плитки: Удержание («N% / клиентов»), Средний чек («₽N / +N% к прошл.»), Рефералов («N / за 3 мес»), Сертификатов («₽N / продано»);
- «Распределение по уровням» — полосы: Серебро («до ₽10K»), Золото («₽10K–50K»), Платина («от ₽50K») с количеством клиентов; итог «Всего клиентов в программе: N».

- «Сравнение по выручке» — полосы по тренерам с суммами.

### 11.5 Вкладка «По услугам»

Вкладка содержит:
- карточки услуг: мини-диаграмма доли %, число занятий, выручка, тренд-бейдж (±%);

### 11.6 Вкладка «Все»

Вкладка содержит:
- 3 сводные карточки: «Клиенты» (Всего, Активных, Новых, Отток), «Занятия» (Проведено, Отменено, Ср. чел., Загрузка), «Финансы» (Выручка, Расходы, Прибыль, Маржа);

Вкладка содержит:
- подсказку «Этот текст определяет личность агента…»;
- большое текстовое поле (по умолчанию: «Ты — вежливый ассистент студии Velora. Отвечай кратко, по делу, помогай клиентам с записью и вопросами об услугах.»);
- счётчик «N / 2000 символов».

### 12.4 Связь с остальным продуктом

ИИ доступен из трёх мест: этот раздел (полный чат + агенты), глобальная AI-панель на любой странице (1.3), AI-строка в верхней панели (1.2.1). История диалогов общая.

## 13. Уведомления

Доступ: только владелец.

### 13.1 Каналы доставки (левая колонка)

Владелец может включать/выключать каждый канал тумблером.

Колонка содержит 6 карточек (иконка, название, подпись, тумблер):
- Telegram — бот студии (по умолчанию вкл);
- Instagram — Direct сообщения (выкл);
- WhatsApp — номер студии (вкл);
- Email — адрес студии (вкл);
- SMS — через SMS-провайдера (выкл);
- Push — мобильное приложение (выкл);
- внизу — совет «Настройте отдельно для каждой роли».

### 13.2 Получатели

Владелец может выбрать получателя: «Клиент» / «Тренер» / «Администратор» / «Владелец».

Карточка получателя содержит: иконку, название, счётчик «N активных» (число событий хотя бы с одним активным каналом).

### 13.3 Матрица «событие × канал»

Владелец может включать/выключать галочки на пересечении события и канала.
Владелец может включить всё кнопкой «Активировать всё» и снять всё кнопкой «Снять все галочки».
Владелец может сохранить изменения кнопкой «Сохранить» (активна только при несохранённых изменениях) или отменить их.

Матрица содержит:
- шапку: иконка роли, «Сценарии для: {Роль}», подпись «Настройте каналы для N системных триггеров»;

- строки — события выбранной роли;
- галочки в цвете канала;
- футер: «Активных триггеров: N из M», кнопки «Отмена» / «Сохранить» / «Активировать всё» / «Снять все галочки»;
- пустое состояние (все каналы выключены): «Нет активных каналов» / «Включите хотя бы один канал доставки в панели слева».

### 13.4 Списки событий по получателям

Клиент (10): подтверждение записи; напоминание о занятии (за 24 ч и за 2 ч); отмена занятия (тренером/студией); успешная оплата (квитанция); осталось мало занятий (1–2 визита); абонемент истекает (за 3 дня); день рождения (поздравление + оффер); запрос отзыва после посещения; возврат средств; задолженность по оплате.

Тренер (8): новая запись к тренеру (персональная); отмена записи клиентом (менее чем за 2 ч); напоминание о занятии (за 1 ч); список участников группы (за 30 мин); изменение в расписании (администратор изменил слот/локацию); начисление зарплаты (еженедельно); новый отзыв о тренере; день рождения клиента.

Администратор (10): новая онлайн-запись (сайт/виджет); отмена в последний момент (менее чем за 1 ч); новый клиент в системе; оплата получена; задолженность клиента (3+ дней); абонементы на исходе (≤1 занятие); конфликт расписания; отчёт за день; попытка входа с нового устройства; возврат средств.

Владелец (9): ежедневная сводка (выручка/записи/посещаемость); еженедельный отчёт (KPI: рост, LTV, конверсия); крупный платёж (свыше порога); аномалия в данных (падение записей/выручки); новый сотрудник добавлен; продление тарифа Velora; изменение прав доступа; достигнут KPI (цель месяца); экспорт данных (кто-то выгрузил базу).

💡 предложение (к обсуждению): при первом входе предлагать готовый набор «Рекомендуемые уведомления» одним кликом (разумные значения уже включены), а матрицу оставить как «расширенную настройку».

## 14. Настройки

Доступ: владелец — всё; администратор и тренер — только персональные вкладки (Внешний вид, Уведомления).

Слева — меню «Настройки системы» из 9 пунктов; внизу — кнопка «Сменить CRM» (выход к выбору пространства).

### 14.1 Основные

Владелец может загрузить/заменить логотип студии.
Владелец может изменить данные компании и региональные настройки.
Владелец может сбросить изменения кнопкой «Сбросить» и сохранить кнопкой «Сохранить».

Вкладка содержит:
- блок «Данные компании»: загрузчик логотипа (превью с редактированием), поля Название компании, Описание, Телефон, Email, Сайт, Адрес;
- блок «Язык и регион»: селекты Часовой пояс, Валюта (₽ / $ / € / ₸), Язык интерфейса, Формат даты (ДД.ММ.ГГГГ / ММ.ДД.ГГГГ / ГГГГ-ММ-ДД), Первый день недели (Понедельник / Воскресенье).

### 14.2 Рабочие часы

Владелец может включить/выключить каждый день недели и задать время «с» и «до».
Владелец может сохранить кнопкой «Сохранить расписание» (→ «Расписание обновлено»).

Вкладка содержит:
- 7 строк дней: тумблер + два тайм-селекта;
- пояснение: указанные часы — официальное время работы студии.

### 14.3 Внешний вид

Пользователь может выбрать тему: «Светлая» / «Тёмная» / «Системная» (карточки-превью).
Пользователь может выбрать акцентный цвет из 6 образцов (персиковый по умолчанию).

Пояснение: цвет применяется к кнопкам, ссылкам и активным элементам.

### 14.4 Уведомления (личные)

Пользователь может включать/выключать личные оповещения.
Пользователь может задать основной и резервный email для уведомлений.

Вкладка содержит:
- тумблеры: «Email-уведомления» (вкл; записи, отмены, платежи), «SMS-оповещения» (выкл), «Push-уведомления» (вкл), «Маркетинговые рассылки» (выкл);
- поля «Основной email» и «Резервный email»;
- кнопку «Сохранить» (→ «Настройки уведомлений сохранены»).

### 14.5 Команда

Владелец может добавить сотрудника кнопкой «+ Добавить сотрудника» (тот же мастер, что в разделе Сотрудники).
Владелец может открыть редактирование сотрудника кнопкой «Изменить».

Вкладка содержит:
- список сотрудников: аватар, имя, email, бейдж роли, статус, кнопка «Изменить»;
- кнопку «+ Добавить сотрудника».

💡 в MVP матрица настраиваемых прав по ролям здесь скрывается: действует модель «три фиксированные роли» (см. «Целевая аудитория» в CLAUDE.md). Существующая матрица прав вернётся позже как фича тарифа Business.

### 14.6 Подписка

Владелец может видеть сводку текущего тарифа.
Владелец может перейти в раздел «Тариф и оплата».

Вкладка содержит:
- сводку: название тарифа, статус, срок, ключевые лимиты;
- кнопку перехода в «Тариф и оплата».

💡 предложение: убрать из этой вкладки собственные прайсы и формы оплаты — сейчас здесь цены, противоречащие каноническим тарифам (Старт 990 / Pro 2490 / Business 5990); всё управление тарифом должно жить в одном месте — разделе «Тариф и оплата».

### 14.7 Безопасность

Владелец может включить двухфакторную аутентификацию (SMS/приложение-аутентификатор).
Владелец может перейти к смене пароля кнопкой «Сменить пароль».
Владелец может завершить любую чужую сессию кнопкой «Завершить» (→ «Сессия завершена»).
Владелец может отозвать ключ интеграции кнопкой «Отозвать».
Владелец может создать новый ключ: «Название интеграции» + «Сгенерировать ключ» (→ «Новый API ключ сгенерирован»).

Вкладка содержит:
- тумблер двухфакторной аутентификации с бейджем «Активна» / «Отключена»;
- блок «Пароль администратора»: дата последней смены + кнопка «Сменить пароль»;
- «Активные сессии»: бейдж «N устройства», список (устройство, браузер, город; текущая помечена «Текущая»);
- «API токены»: список с маскированными ключами и датами, кнопка «Отозвать», кнопка «+ Создать новый токен»;
- «Опасную зону»: «Экспорт всех данных» (чекбоксы: база клиентов / финансовые транзакции / медиа и фото; «Начать выгрузку» → архив на email), «Очистить базу данных» и «Удалить аккаунт компании».

При очистке базы или удалении аккаунта:
- открывается отдельное окно подтверждения с предупреждением о безвозвратности и кнопками «Отмена» / «Стереть данные» или «Удалить навсегда».

### 14.8 Интеграции

Владелец может подключить/отключить каждую из 6 интеграций.

Каждая карточка содержит: статус «Подключено» / «Не активно», кнопку «Настроить» / «Подключить», свои поля, у подключённых — кнопку «Отключить интеграцию»:
- WhatsApp Business: номер телефона, служебный адрес приёма событий;
- Telegram-бот: токен, приветственное сообщение;
- Instagram Direct: бизнес-аккаунт;
- Google Calendar: название календаря, режим синхронизации (Двусторонняя / Только экспорт / Только импорт);
- 1С: Предприятие: адрес шлюза, логин;
- ЮKassa: идентификатор магазина, тумблер «Тестовый режим оплаты».

После настройки — уведомление «Интеграция … настроена» / «… отключена».

### 14.9 Данные

Владелец может видеть занятость хранилища.
Владелец может включить автоматическое резервное копирование и задать срок хранения (14 / 30 / 60 / 90 дней).
Владелец может скачать последний бэкап и сгенерировать бэкап сейчас.
Владелец может экспортировать данные плитками экспорта.

Вкладка содержит:
- «Хранилище данных»: прогресс «X из Y ГБ» с разбивкой (Клиенты / Медиа-файлы / Документы);
- блок бэкапов: тумблер «Автоматическое резервное копирование» (ежедневно ночью), «Последняя успешная копия — дата · размер», селектор срока хранения, кнопки «Скачать последний бэкап» и «Сгенерировать бэкап сейчас»;
- «Экспорт отчётов и списков» — 4 плитки: Полный список клиентов (CSV/XLSX), История журналов и записей (CSV/PDF), Финансовый отчёт и касса (XLSX), Сводная AI-аналитика студии (PDF).

### 14.10 Переключатель студий

Пользователь может войти в другую свою студию кликом по карточке (→ «Вход выполнен: …»).

Экран содержит:

- блок «Создать новую CRM-систему»: поле «Название студии», выбор темы (Светлая / Тёмная), кнопка создания.

## 15. Тариф и оплата

Доступ: только владелец.

### 15.1 Шапка

Владелец может видеть текущий тариф и сводные показатели.

Шапка содержит:
- плашку «ТЕКУЩИЙ ТАРИФ: {название}» (статус, дата окончания, цена/мес);
- 4 счётчика с анимацией набора: «Потрачено всего», «Месяцев с нами», «Сэкономлено», «Следующее списание»;
- вкладки: «Тарифы и планы» / «История платежей» / «Способ оплаты».

### 15.2 Вкладка «Тарифы и планы»

Владелец может выбрать тариф (у текущего кнопка неактивна — «Текущий план»).
Владелец может выбрать период оплаты: 1 месяц / 6 месяцев (−20%) / 1 год (−30%, «Лучший выбор») / 2 года (−40%).
Владелец может подтвердить смену тарифа в окне апгрейда.

Тарифы:
- «Старт» — 990 ₽/мес: до 3 сотрудников, до 100 клиентов, онлайн-запись, базовый календарь; без аналитики, API-доступа, лояльности, white-label;
- «Pro» — 2490 ₽/мес: до 20 сотрудников, неограниченно клиентов, полная аналитика, лояльность и CRM, Telegram-уведомления, приоритетная поддержка; без API-доступа и white-label;
- «Business» — 5990 ₽/мес: неограниченно всё, white-label, API-доступ, мультифилиалы, выделенный менеджер, кастомные интеграции, SLA 99.9%, обучение команды.

При выборе периода больше месяца:
- цены пересчитываются, показывается зачёркнутая базовая цена и бейдж «Скидка N% активна».

Вкладка также содержит:
- иллюстрацию экономии: зачёркнутая полная цена, сумма к оплате «За N месяцев», плашка «−N% · ₽X экономия», сравнение «Это как N месяца бесплатно»;
- «График платежей»: точки будущих платежей, «Итого к оплате ₽X», кнопка «Оплатить N месяцев»;
- блок «Модель оплаты» — 3 варианта: «Фиксированная подписка» (по умолчанию, бейдж «Популярно»), «% с онлайн-платежей» (калькулятор: слайдер оборота + ставка 2–8%, расчёт итогового платежа), «Фикс + % комбо». 💡 предложение: для MVP оставить только фиксированную подписку — процентные модели усложняют выбор и биллинг; обсудить;
- блок доверия: отзывы клиентов Velora с оценками ★★★★★.

Окно апгрейда содержит:
- заголовок «Переход на {План}»;
- сводку: Тариф / Период / Скидка (−N%, −₽X) / Итого ₽X;
- кнопки «Подтвердить и оплатить» / «Отмена»;
- подпись «Защищено PCI DSS · Возврат в течение 7 дней».

### 15.3 Вкладка «История платежей»

Владелец может экспортировать историю кнопкой «Экспорт CSV».
Владелец может скачать квитанцию PDF у каждой строки.

Вкладка содержит:
- график «Динамика расходов за 2 года»: столбцы по 24 месяцам с тултипами, итоговая сумма трат;
- таблицу: Дата / Описание / Сумма / Чек.

### 15.4 Вкладка «Способ оплаты»

Владелец может привязать новую карту.
Владелец может настроить автоплатёж тумблерами.

Вкладка содержит:
- визуальную банковскую карту: маскированный номер, держатель, срок; бейдж «Основная»;
- кнопку «Привязать новую банковскую карту» → форма: Номер карты (автоформатирование), Имя держателя (латиницей), Срок MM/YY, CVC (скрытый); кнопки «Отмена» / «Сохранить карту»; проверка корректности номера;
- блок «Стандарты защиты данных»: PCI DSS Level 1, 3D Secure, «данные карт не хранятся», «уведомления за 3 дня»;
- «Настройки автоплатежа» — 4 тумблера: «Автоматическое продление» (вкл), «Email-уведомления» (вкл), «Уведомить за 3 дня» (вкл), «SMS-оповещение» (выкл).

## 17. Клиентское мини-приложение (вторая очередь — после MVP CRM)

Отдельное приложение для клиентов студии; клиент попадает в него по ссылке из Telegram, Instagram или с сайта студии. Брендинг (цвет, логотип, язык) и правила записи задаёт студия в разделе «Онлайн-запись».

### 17.1 Быстрая запись без входа

Клиент может записаться на занятие с главной страницы студии без входа в кабинет.

1. выбор услуги (название, длительность, свободные места, цена);
2. выбор даты и времени из доступных слотов;
3. подтверждение — «Запись подтверждена» со сводкой (услуга, время, стоимость).

### 17.2 Личный кабинет клиента

Клиент может видеть свои записи, записываться и отменять запись (по правилам студии: до какого времени можно отменить).
Клиент может видеть свой абонемент: остаток занятий и срок действия.
Клиент может видеть карту лояльности: баллы и уровень.
Клиент может видеть историю посещений.
Клиент может оставлять отзывы и оценки занятий.
