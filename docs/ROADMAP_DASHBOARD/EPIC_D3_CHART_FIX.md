# EPIC D3 — График: одиночный бар и масштаб оси X

**Цель:** период с одной точкой данных рисует **компактный бар у левого края**, а
не полосу во всю ширину карточки. Ось X масштабируется по числу точек и остаётся
читаемой на всех периодах (1…13 баров).

**Зависимости:** нет, можно делать параллельно чему угодно. **Оценка ~1:00.**
Только фронт (CSS + небольшой TSX).

---

## Диагноз: это не Recharts и не свечи

Аудит говорит «свеча». В коде страницы **нет** ни Recharts, ни свечного графика —
это CSS-flex-бары:

- Рендер: `Overview/components/widgets/AnalyticsChart.tsx:56-78` —
  `<div id="dash-chart"> → <div class="chart-bars"> → N × <div class="bar">`,
  под ними `<div class="chart-labels"> → N × <div class="chart-label">`.
- Контейнер: `App.css:976` — `#dash-chart { height: 300px; display: flex; flex-direction: column; justify-content: flex-end }`.
- Полоса баров: `App.css:992` — `.chart-bars { display: flex; align-items: flex-end; height: 100%; gap: 10px }`.
- **Причина дефекта:** `App.css:1000` — `.bar { flex: 1 }`. `flex-grow: 1` заставляет
  единственный бар занять **всю** свободную ширину. `.chart-label` (`App.css:1042`)
  страдает тем же.

Recharts в проекте есть (используется Отчётами и Финансами), но тащить его сюда
ради одного правила ширины — замена двух строк CSS на новый слой рендера. Не делаем.

## Frontend UI & Components

### Задача 1. CSS: бар растёт, но не тянется (~0:30)

**Файл:** `front/src/App.css`.

```css
.chart-bars {
  display: flex;
  align-items: flex-end;
  justify-content: flex-start;   /* ← бары от левого края; при 1 точке не центрируем */
  height: 100%;
  gap: 10px;
  margin-top: 35px;
}

.bar {
  flex: 0 1 56px;                /* ← было flex: 1. grow=0 — вот и весь фикс */
  min-width: 18px;               /* 13 баров в узкой колонке всё ещё различимы */
  max-width: 56px;
  border-radius: 6px 6px 0 0;
  background: rgba(252,174,145,0.2);
  transition: all 0.4s cubic-bezier(0.34,1.56,0.64,1);
  position: relative;
  cursor: pointer;
}

.chart-label {
  flex: 0 1 56px;                /* ← синхронно с .bar, иначе подпись «уплывёт» */
  min-width: 18px;
  max-width: 56px;
  text-align: center;
  font-size: 10px;
  color: var(--text3);
  font-weight: 600;
}
```

**Инвариант, который нельзя нарушать:** `.bar` и `.chart-label` обязаны иметь
одинаковые `flex-basis` / `min-width` / `max-width`, а `.chart-bars` и
`.chart-labels` — одинаковый `gap` (сейчас `10px` в обоих, `App.css:996` и `:1038`).
Иначе колонки перестают совпадать с подписями. Комментарий об этом в CSS уже стоит —
сохранить.

> **Побочные эффекты изменения глобального CSS проверены:** классы
> `.chart-bars`/`.bar`/`.chart-label` используются ровно в двух местах —
> `AnalyticsChart.tsx` (наша страница) и `front/src/pages/Dashboard.tsx`
> (легаси-макет на моках: 876 строк, не импортируется ни одним роутом —
> `grep -rn "pages/Dashboard'" front/src` → пусто). Регрессий в живых страницах нет.

### Задача 2. TSX: отдельная ветка для одной точки не нужна (~0:15)

**Файл:** `AnalyticsChart.tsx`. После Задачи 1 специальный кейс `series.length === 1`
не требуется — `flex-grow: 0` + `justify-content: flex-start` уже дают компактный
первый бар. Проверить только два места:

1. **Высота.** `Math.max(4, (p.value / max) * 100)` при одной точке даёт
   `max === value` → бар 100 % высоты. Это корректно: единственная точка —
   собственный эталон. Оставляем.
2. **Ключ и анимация.** `key={p.period}` уникален (бакеты не повторяются).
   Stagger-задержки `.bar:nth-child(1…8)` (`App.css:1258`) покрывают 8 баров; при
   `year` (12-13 точек) хвост стартует без задержки — визуально приемлемо, но если
   хочется ровного каскада, заменить 8 правил одной строкой:
   ```css
   .bar { animation-delay: calc(var(--i, 0) * 0.05s); }
   ```
   с `style={{ '--i': i } as CSSProperties}` в `AnalyticsChart.tsx`. Опционально.

### Задача 3. Масштаб оси X на краях диапазона (~0:15)

Ширина колонки виджета — половина `grid-2` (график делит ряд с виджетом задач).
Считаем худший случай: 13 баров × 56 px + 12 × 10 px gap = **848 px** — шире
колонки на типовом ноутбуке. Поведение по умолчанию (`flex-shrink: 1` +
`min-width: 18px`) сожмёт бары до 18 px и они влезут, но при совсем узком
контейнере начнут выдавливаться.

Страховка одной строкой в `App.css`:

```css
#dash-chart { overflow-x: auto; overscroll-behavior-x: contain; }
```

Горизонтальный скролл вместо схлопывания. `overflow-y` не трогаем — тултипы
баров (`.bar-tooltip`, `top: -28px`) уходят выше контейнера и не должны обрезаться;
при необходимости поднять `padding-top` у `#dash-chart` на 28 px.

**Матрица проверки** (все — на реальных данных, не на моках):

| Период | Точек | Ожидание |
|---|---|---|
| Любой, пустая студия | 0 | `t('chart.noData')`, баров нет |
| `Мес.`, данные за один месяц | 1 | один бар ~56 px у левого края + подпись под ним |
| `Нед.` | 2-8 | равные бары, подписи `дд/мм` без наложения |
| `Год` | 12-13 | подписи месяцев читаемы; при нехватке ширины — скролл, а не каша |
| Метрика «Удержание» | — | `t('chart.retentionNA')`, запрос ряда не уходит (`enabled: false`) |
