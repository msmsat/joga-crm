# EPIC D3 — Фикс графика: одиночный бар и масштаб оси X

**Цель:** когда за период есть только одна точка (или мало точек), бар не
растягивается на весь холст — он компактный, прижат к левому краю; ось X
масштабируется по числу точек.

**Зависимости:** нет. **Оценка: ~1:00.** Только фронт (CSS + TSX).

---

## Диагноз (важно — это НЕ Recharts/свечи)

Аудит говорит «свеча». В коде графика свечей и Recharts **нет**. Это
CSS-flex-бары:

- Рендер: [`AnalyticsChart.tsx:57-74`](../../front/src/pages/dashboard/Overview/components/widgets/AnalyticsChart.tsx#L57) —
  `<div className="chart-bars">` + `<div className="bar">` на точку.
- Причина растяжения: [`App.css:1000`](../../front/src/App.css#L1000) —
  `.bar { flex: 1 }`. При одной точке `flex:1` занимает всю строку → «на весь
  холст». `.chart-bars` — `display:flex; gap:10px` ([`App.css:992`](../../front/src/App.css#L992)).

## Frontend — Задача 1. CSS: фиксированная ширина бара (~0:30)

**Файл:** [`front/src/App.css`](../../front/src/App.css#L1000).

Заменить «резиновую» ширину на фиксированную с потолком, чтобы N баров не зависели
от ширины контейнера:

```css
.chart-bars {
  display: flex;
  align-items: flex-end;
  justify-content: flex-start;   /* ← бары от левого края, не по центру/растяжкой */
  height: 100%;
  gap: 10px;
  margin-top: 35px;
}

.bar {
  flex: 0 1 48px;                /* ← было flex:1. Растёт до 48px, но не тянется на весь холст */
  min-width: 20px;
  max-width: 48px;
  border-radius: 6px 6px 0 0;
  /* остальное без изменений */
}
```

`.chart-labels` / `.chart-label` ([`App.css:1036`](../../front/src/App.css#L1036))
сейчас `flex:1` — привести к той же ширине, иначе подпись «уплывёт» от бара:

```css
.chart-label { flex: 0 1 48px; min-width: 20px; max-width: 48px; text-align: center; }
```

> Двумя правилами. Никаких JS-вычислений ширины, никакого Recharts (ponytail:
> CSS уже умеет «расти-но-не-тянуться» через `flex-grow:0`).

## Frontend — Задача 2. TSX: без специального «одиночного» кейса (~0:15)

**Файл:** `AnalyticsChart.tsx`. После CSS-фикса отдельная ветка для `length===1`
**не нужна** — `justify-content:flex-start` + фикс-ширина уже дают компактный
первый бар. Проверить только высоту:

- `Math.max(4, (p.value / max) * 100)` при одной точке даёт `max===value` → бар
  100% высоты. Это корректно (единственная точка = эталон). Оставить.
- Убедиться, что `key={p.period}` уникален и при 1 точке не ломает анимацию
  `barGrow` ([`App.css:1257`](../../front/src/App.css#L1257) — nth-child задержки
  до 8, при 1 баре просто первая).

## Frontend — Задача 3. Масштаб оси X (~0:15)

Число баров = число точек в `series` (уже так). Проверить крайние периоды:

- `week` → до 8 недель, `month` → до 8, `year` → до 12 точек
  ([`useOverviewData.ts:34`](../../front/src/pages/dashboard/Overview/hooks/useOverviewData.ts#L34)).
- При 12 барах × 48px + gap не должно быть переполнения — если контейнер уже
  суммы, добавить на `#dash-chart` `overflow-x:auto` (горизонтальный скролл
  вместо сжатия). Одной строкой в `App.css`.

## Definition of Done

- Период с 1 точкой: бар ~48px шириной, прижат влево, подпись под ним. **Не** на
  весь холст.
- Период с 12 точками: бары равной ширины, ось X читаема, без наложения подписей.
- Смена метрики/периода не ломает выравнивание.
- `cd front && npm run build` — чисто; визуальная проверка обоих кейсов.
