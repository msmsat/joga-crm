## Важно понимать про старт — бэкенд-ядро готово, не переписывать

- **CRUD целиком есть** по всем сущностям (`back/routers/finances/`): счета
  (`accounts.py` — GET/POST/PATCH/DELETE, `daily_change` считается на лету), операции
  (`operations.py` — GET с фильтрами/поиском/пагинацией, POST двигает баланс счёта,
  начисляет баллы и шлёт уведомление c4, DELETE откатывает баланс), контрагенты
  (`counterparties.py` — CRUD, balance/deals_count на лету из операций), документы
  (`documents.py` — CRUD), цели (`goals.py` — CRUD), зарплаты (`salary.py` — расчёт
  по ставке fixed/hourly/percent + выплата).
- **Авто-леджер уже почти собран.** Деньги падают в «Операции» из: продажи абонемента
  (`back/routers/clients/subscriptions.py:87`, категория «Абонементы»), пополнения
  депозита (`back/routers/clients/loyalty.py:149`, «Депозит»), выплаты зарплаты
  (`back/routers/finances/salary.py:141`, расход «Зарплата»). **Единственная дыра —
  продажа сертификата** (`back/routers/loyalty/certificates.py:51` операцию не создаёт).
- **Авто-цели уже работают на бэке:** `_auto_amount` (`back/routers/finances/goals.py:16`)
  суммирует доходы категории от `created_at` до дедлайна. Не хватает только выбора
  категории из реальных (на фронте — свободный текстовый инпут, `GoalsTab.tsx:214`)
  и расходных целей.
- **Отчётные агрегаты уже написаны:** `GET /analytics/summary`, `/series`, `/trainers`,
  `/services` (`back/routers/analytics/reports.py:119,162,218,282`) — считают
  выручку/расходы/прибыль/тренды по реальным операциям. Вкладка «Отчёты» их просто не зовёт.
- **Модели под шлюзы уже в БД:** `OnlineChannel`, `PaymentMethodConfig`,
  `OnlineChannelMetric` (`back/models/finances.py:88,103,158`) — роутеров под них нет,
  обе вкладки живут на `ONLINE_CHANNELS_DATA`/`PAYMENT_METHODS_DATA` (`constants.ts:5,12`).
- **Словарь уже создан:** `front/src/locales/{ru,en}/finances.json` существуют и **пустые**;
  ни один файл вкладок не импортирует `useTranslation`. Регистрация — по образцу
  `clients` в `front/src/i18n.ts`.
- **Инфраструктура с прошлых аудитов готова:** TanStack Query (`front/src/api/queryClient.ts`,
  `queryKeys.ts` — в комментарии прямо «очередь миграции: … Финансы»), `errorMessage.ts`,
  общий Toast **уже подключён** (`Finances.tsx:31`) — проблема не в тостах, а в том,
  что половина из них врёт («Настройки шлюза обновлены» при чисто локальном стейте).
- **Валюта:** ₽ вшит в `fmt()` (`front/src/pages/dashboard/Finances/constants.ts:3`) и
  разлетается по всем вкладкам. Готовые кирпичи: хук `useStudioCurrency`
  (`front/src/pages/dashboard/Catalog/hooks/useStudioCurrency.ts`, кэш-квери) и
  `getCurrencySymbol` (`front/src/components/UI.tsx:482`).
- **Мелкие дыры бэка, из которых вырастают задачи:** `OperationRead` не отдаёт
  `counterparty_id`/`trainer_id`, хотя в БД они есть (`back/schemas/finances/operations.py:7`);
  PATCH операции нет вовсе; `SalaryRow` не отдаёт часы/выручку (уже посчитаны в
  `_sessions_and_hours`, `salary.py:19`); истории выплат эндпоинта нет; ставку сотрудника
  уже умеет править `PATCH /staff/{id}` (`back/routers/staff/profiles.py:376`);
  у `FinDocument` есть `file_url` (`back/models/finances.py:81`), но файлы никто не грузит.
- Локальные UI-примитивы вкладок (`Btn`, `Toggle`, `Badge`, `ConfirmModal` в
  `Finances/components/ui/`) **не трогаем** — это не аудит UI-кита. Новое общее
  (InfoHint) — в `front/src/components/ui/` по правилам кита.

**Обозначения сложности в бэклогах:** 🟢 простая · 🟡 средняя · 🔴 сложная, делать внимательно.

## Финальная задача — Сквозной прогон и фиксация в STATUS · 🟡 · 1:00

Делается после всех эпиков.

**Что конкретно сделать:**
1. Чек-лист: продать абонемент из Клиентов → операция «Абонементы» видна в Операциях
   без F5, баланс счёта и цель категории «Абонементы» выросли; выпустить сертификат →
   операция «Сертификаты»; выплатить зарплату → расход «Зарплата» + история выплат
   в карточке; правка суммы операции пересчитала баланс счёта; контрагент создаётся
   из формы расхода не выходя из неё; Stripe без ключей = «Не подключён», с ключами =
   «Подключён»; «Методы оплаты» показывают реальные суммы по нал/карте/QR/Stripe/Fondy;
   «Отчёты» совпадают с /analytics/summary, пустой период → «Нет данных за этот период»;
   экспорт CSV скачивает файл с фильтрами; документ с файлом скачивается, поповер
   «три точки» не улетает при скролле; en-локаль — ни одной русской строки; валюта $
   у студии с $ — нигде нет ₽; каждая вкладка имеет i-кнопку с поповером.
2. `npm run build && npm run lint` — чисто; бэкенд-тесты зелёные.
3. Обновить `docs/TZ/STATUS.md` (раздел Финансы) и проставить статусы эпиков в этом файле.

**Готово, когда:** чек-лист пройден, статус зафиксирован.

## Definition of Done роадмапа

- Ни одного выдуманного числа на странице: моки `ONLINE_CHANNELS_DATA`,
  `PAYMENT_METHODS_DATA`, `LINE_DATA`, хардкод-метрики Отчётов — удалены из кода.
- Ни одной русской строки-хардкода; валюта — символ студии; `fmt()` с ₽ не существует.
- Любая мутация обновляет все затронутые вкладки через инвалидацию кэша — ручных
  `setState`-патчей списков и F5 нет.
- Ни одной «мёртвой» кнопки: экспорт отдаёт CSV, документы скачиваются реальным файлом,
  либо кнопка удалена; тосты не сообщают об успехе локального стейта.
- Любая финансовая активность продукта (абонемент, депозит, сертификат, зарплата,
  ручная операция) видна в «Операциях» и в отчётных агрегатах.
- «Онлайн-платежи» = только Stripe и Fondy с правдивым статусом; «Методы оплаты» =
  только реальная статистика сумм по методам.
- Общие компоненты переиспользованы, ничего app-wide не построено; сборка и линт чистые.
