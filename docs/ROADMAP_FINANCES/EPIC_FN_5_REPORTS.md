### 5.1 Экспорт операций в CSV · 🟡 · 1:30 · ✅ СДЕЛАНО
**Простыми словами:** «Экспорт PDF» в Отчётах — `showToast('Отчёт экспортируется...')`
(`ReportsTab.tsx:396`) и ничего. Делаем настоящий CSV по операциям (то, что реально
нужно бухгалтерии в MVP); PDF не обещаем — кнопку переименовываем.
**Что конкретно сделать:**
1. `back/routers/finances/operations.py` → `GET /operations/export` (те же
   query-фильтры, что у списка; роут выше `{operation_id}`): `csv.writer` в `StringIO` →
   `StreamingResponse(media_type="text/csv")` с `Content-Disposition: attachment;
   filename=operations_<from>_<to>.csv`; колонки: дата, тип, название, сумма, категория,
   метод, счёт, клиент, контрагент (имена, не id — один запрос со `join`/словарями).
   BOM `﻿` в начало — иначе Excel ломает кириллицу.
2. `front/src/api/finances/finances.api.ts` → `exportOperations(filters)`: fetch с
   токеном → `blob` → ссылка-скачивание (образец, если есть в репо экспорт — иначе
   `URL.createObjectURL` + `a.click()`).
3. `ReportsTab.tsx:396` → кнопка «Экспорт CSV» зовёт экспорт с текущим периодом вкладки;
   такую же кнопку добавить в тулбар `OperationsTab.tsx` (экспорт с активными фильтрами).
**Готово, когда:** файл скачивается, открывается в Excel с кириллицей, состав строк
совпадает с фильтрами на экране.

### 5.2 Документы: реальные файлы — загрузка и скачивание · 🔴 · 2:30 · ✅ СДЕЛАНО
**Простыми словами:** зона «Нажмите или перетащите файл» — декорация
(`DocumentsTab.tsx:159-169`, клик → тост «Выбор файла...»), кнопка «Скачать» — тост
«Скачивание файла...» (`:225`). При этом модель готова: `file_url`, `file_ext`
(`back/models/finances.py:80-81`). Правило аудита: кнопка либо работает, либо удаляется —
делаем работу.
**Что конкретно сделать:**
1. `back/routers/finances/documents.py` →
   - `POST /documents/{doc_id}/file` (`UploadFile`): сохранить в `back/uploads/docs/
     {studio_id}/{doc_id}.{ext}` (создать папку; в `.gitignore` добавить `back/uploads/`),
     записать `file_url` и `file_ext` по реальному расширению; лимит 15 МБ и белый список
     ext (pdf/docx/xlsx) → иначе 400;
   - `GET /documents/{doc_id}/file` → `FileResponse` c исходным именем; нет файла → 404.
     Оба — только своя студия (готовый `_get_or_404`).
2. `front/src/api/finances/finances.api.ts` → `uploadDocumentFile(id, file)` (FormData),
   `downloadDocumentFile(id)` (blob-скачивание как в 5.1).
3. `DocumentsTab.tsx` → дроп-зона: скрытый `<input type="file" accept=".pdf,.docx,.xlsx">`
   + обработчики dragover/drop; выбранный файл подставляет название документа; сабмит =
   `createDocument` → `uploadDocumentFile`. «Скачать» активна только у документов с
   файлом (поле `has_file`/`file_url` добавить в `FinDocumentRead`), у остальных
   кнопку не рендерить.
**Готово, когда:** PDF загружается перетаскиванием, скачивается тем же файлом; документ
без файла кнопки «Скачать» не имеет.

### 5.3 Документы: поповер «три точки» не улетает · 🟢 · 0:45 · ✅ СДЕЛАНО
**Простыми словами:** поповер позиционируется `position: fixed` по координатам кнопки,
снятым в момент клика (`DocumentsTab.tsx:226,237`), а контент вкладки скроллится в
собственном контейнере (`Finances.tsx:77-88`) — прокрутил список при открытом поповере,
и он завис в воздухе / уехал от кнопки.
**Что конкретно сделать:** `DocumentsTab.tsx` → якорить поповер к строке: обёртке кнопок
`position: relative`, поповеру `position: absolute; right: 0; top: calc(100% + 6px)`
(вверх — `bottom: calc(100% + 6px)` по той же проверке `up`); `fixed`-оверлей для
клика-мимо оставить. Замер `getBoundingClientRect` для выбора вверх/вниз оставить.
**Готово, когда:** поповер открывается вплотную к точкам и едет вместе со строкой при
скролле; у последних строк раскрывается вверх.
