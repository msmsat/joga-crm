**Вне рамок (кандидаты без бэклогов):** вебхуки входящих сообщений TG/IG-автоответчиков и
AI-чат Журнала на общем `useAssistant` (из аудита ROADMAP_AI 2026-07-22 — первым нужен живой

сущность (вкладка «События» Отчётов); вкладки Финансов «Онлайн-платежи» и «Способы оплаты»
(ждут контура клиентских оплат); ручной прогон Fondy на тестовом мерчанте (исполнение — с
деплоем в эпике 7); FIFO-разбивка сгорания баллов по месяцам (сейчас агрегатная формула,
V5-7 Блок 3 — эквивалентна FIFO, но без помесячного брейкдауна); автопродление абонемента
с карты (сейчас только за счёт депозита клиента, V5-7 Блок 4.2 — эквайринга в продукте ещё
нет); реферальная ссылка в клиентском мини-приложении (сейчас код только в CRM, V5-7 Блок 2);
скидки, привязанные к конкретным услугам (`applies_to_all_services`, поле есть в БД, тумблер
скрыт в V5-7 Блок 5 — нужна отдельная модель привязки); витрина скидок в мини-приложении
(`visible_in_cabinet`, поле есть в БД, тумблер скрыт в V5-7 Блок 5 — ждёт само мини-приложение);
Отчёты/Клиенты: когортное удержание (матрица месяц регистрации × месяц визита); данные уже
достаточны (`registration_date` + `reservations`), нужен только эндпоинт + грид. Источник:
ТЗ Reports, «второй этап» (EPIC_R3_CLIENTS.md, задача 4).

**Невозможные сейчас события уведомлений (Уведомления, N-4)** — матрица на фронте их
сохраняет (галочки работают), шаблонов у `notify()` нет — честное поведение до реализации:
- **a7 «Конфликт расписания»** — в проекте нет детектора пересечений занятий; строить его
  ради одного уведомления — не задача модуля уведомлений;
- **a9 «Попытка входа»** — требует трекинга «новое устройство» в auth-флоу (`UserSession`
  пишется, но сравнения устройств между собой нет);
- **o9 «Экспорт данных»** — врежется, когда появится эндпоинт экспорта базы (раздел
  Настройки → Данные сейчас без бэка);
- **t1 «Новая запись к тренеру», t2 «Отмена записи клиентом», t5 «Изменение в
  расписании», t7 «Новый отзыв о тренере»** — шаблонов у `notify()` нет, врезка не
  сделана; ближайшие кандидаты (t1/t2 зеркалят уже готовые c1/c3 в записи/отмене,
  t5 — перенос занятия уже врезан для клиента как c11, для тренера — нет).
